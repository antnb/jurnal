<h1><a name="bookmark2"></a><span class="font2" style="font-weight:bold;"><a name="bookmark3"></a>ABSTRAK</span></h1>
<p><span class="font2">Hingga awal tahun 2005, karakteristik interaksi aplikasi web selalu tergantung pada </span><span class="font2" style="font-style:italic;">event-event</span><span class="font2"> yang diberikan oleh pengguna. Serta, harus menunggu munculnya halaman web secara utuh untuk berpindah dari satu </span><span class="font2" style="font-style:italic;">event</span><span class="font2"> ke </span><span class="font2" style="font-style:italic;">event</span><span class="font2"> lain. Untuk memperbaiki masalah ini, muncul teknik </span><span class="font2" style="font-style:italic;">Asynchronous Javascript and XML</span><span class="font2">(AJAX) yang membuat aplikasi web menjadi mirip aplikasi </span><span class="font2" style="font-style:italic;">desktop</span><span class="font2"> dan tidak terlalu tergantung pada </span><span class="font2" style="font-style:italic;">event</span><span class="font2"> yang diberikan pengguna. Namun, Ajax juga memiliki kekurangan karena kode sumbernya sulit untuk ditulis. Beberapa framework hadir untuk mengatasi kekurangan ini, salah satunya adalah ASP.NET Ajax.</span></p>
<p><span class="font2">Penelitian ini mencoba membuat sebuah aplikasi web Ajax dengan menggunakan ASP.NET Ajax untuk dibandingkan dengan aplikasi web tradisional. Aspek-aspek yang dibandingkan dalam penelitian ini adalah waktu loading dan jumlah data yang dilewatkan. Aplikasi yang dibuat adalah aplikasi web mengenai informasi kepariwisataan di propinsi Daerah Istimewa Yogyakarta.</span></p>
<p><span class="font2">Hasil penelitian menunjukkan bahwa aplikasi web tradisional memiliki waktu loading yang lebih singkat pada saat me-</span><span class="font2" style="font-style:italic;">refresh</span><span class="font2"> suatu halaman karena melewatkan data yang lebih sedikit dibandingkan aplikasi web Ajax dengan Atlas. Namun, untuk mengubah isi suatu halaman, aplikasi web Ajax dengan menggunakan ASP.NET Ajax memiliki waktu loading yang lebih singkat karena melewatkan data lebih sedikit.</span></p>
<p><span class="font2" style="font-weight:bold;">Kata Kunci </span><span class="font2">: </span><span class="font2" style="font-style:italic;">waktu loading, jumlah data, AJAX, ASP.NET Ajax</span></p>
<h1><a name="bookmark4"></a><span class="font2" style="font-weight:bold;"><a name="bookmark5"></a>ABSTRACT</span></h1>
<p><span class="font2">Until the beginning of 2005, the characteristic of the web application interaction always depends on the events given by the user. Moreover, must wait for the occurrance of the whole web page to move from one event to another. To repair this problems, appears the Asynchronous Javascript and XML(AJAX) technique which makes the web application similar to desktop application and not depends too much on the event given by the user. Ajax also be claimed has the shorter loading time. However, Ajax also has some weakness because its source code is hard to be written. Some frameworks appear to overcome this weakness, one of them is ASP.NET Ajax.</span></p>
<p><span class="font2">This research tries to make an Ajax web application with ASP.NET Ajax to compare with the traditional web. The compared aspects in this research are the loading time and the data amount. The web application that made is tourism information in Daerah Istimewa Yogyakarta province.</span></p>
<p><span class="font2">The result shows that the traditional web application has the shorter loading time in a whole page refreshing because it passes less data amount rather than Ajax web application using Atlas. However, to change a web page contents, Ajax web application using the ASP.NET Ajax has the shorter loading time because it passes less data amount.</span></p>
<p><span class="font2" style="font-weight:bold;">Keywords </span><span class="font2">: &nbsp;loading time, data amount, AJAX, ASP.NET Ajax</span></p>
<h1><a name="bookmark6"></a><span class="font2" style="font-weight:bold;"><a name="bookmark7"></a>PENDAHULUAN</span></h1>
<p><span class="font2">Aplikasi berbasis </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> adalah aplikasi yang berkembang sangat dinamis sejak dimunculkan pada awal tahun 1990. Generasi pertama adalah aplikasi </span><span class="font2" style="font-style:italic;">web </span><span class="font2">statis yang dimulai dengan hadirnya HTML sebagai standar penulisan aplikasi berbasis </span><span class="font2" style="font-style:italic;">web</span><span class="font2">. Selang beberapa tahun kemudian aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> dinamis hadir. Namun, karakteristik aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> tetap tidak berubah, yaitu sangat tergantung dari </span><span class="font2" style="font-style:italic;">event-event</span><span class="font2"> yang diberikan oleh pengguna dan harus menunggu munculnya halaman </span><span class="font2" style="font-style:italic;">web </span><span class="font2">secara utuh untuk berpindah </span><span class="font2" style="font-style:italic;">event</span><span class="font2">. Gambar 1 mengilustrasikan interaksi antara </span><span class="font2" style="font-style:italic;">client</span><span class="font2"> dan </span><span class="font2" style="font-style:italic;">server</span><span class="font2"> pada aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> tradisional.</span></p><img src="https://jurnal.harianregional.com/media/2690-1.jpg" alt="" style="width:156pt;height:114pt;">
<p><span class="font2">Gambar 1. Interaksi </span><span class="font2" style="font-style:italic;">client</span><span class="font2"> dan </span><span class="font2" style="font-style:italic;">server </span><span class="font2">pada aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> tradisional Menurut [SMI-06], </span><span class="font2" style="font-style:italic;">Asynchronous JavaScript and XML</span><span class="font2">, atau Ajax mulai populer pada awal tahun 2005 setelah kemunculan Google Maps, Google Gmail dan Flickr. Ajax, meskipun menggunakan &nbsp;&nbsp;&nbsp;&nbsp;teknologi &nbsp;&nbsp;&nbsp;&nbsp;lama,</span></p>
<p><span class="font2">merupakan paradigma baru dalam pembuatan aplikasi berbasis </span><span class="font2" style="font-style:italic;">web</span><span class="font2">. Dengan Ajax, aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> menjadi mirip dengan aplikasi </span><span class="font2" style="font-style:italic;">desktop</span><span class="font2">.</span></p>
<p><span class="font2">Ajax adalah istilah yang diperkenalkan pertama kali oleh Jesse James Garrett dari Adaptive Path. Dalam esainya </span><span class="font2" style="font-style:italic;">“AJAX : A New Approach to Web Applications”</span><span class="font2">, Garrett menyebutkan bahwa jurang antara aplikasi </span><span class="font2" style="font-style:italic;">desktop </span><span class="font2">dengan aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> semakin menipis. Hal tersebut dibuktikan oleh Google dengan munculnya Google Maps dan Google Suggest.</span></p>
<p><span class="font2">Menurut [GAR-05] teknologi-teknologi pembentuk Ajax terdiri dari : (1) HTML dan CSS, untuk lapisan presentasi; (2) </span><span class="font2" style="font-style:italic;">Document Object Model</span><span class="font2">, untuk tampilan dan interaksi dinamis; (3) XML, sebagai format pertukaran data; (4) XMLHttpRequest, untuk mengambil data secara </span><span class="font2" style="font-style:italic;">asynchronous;</span><span class="font2"> (5) Javascript, untuk menggabungkan semua teknologi diatas</span></p>
<p><span class="font2">Ajax adalah kumpulan teknologi-teknologi yang membuat sebuah </span><span class="font2" style="font-style:italic;">browser</span><span class="font2"> bisa berkomunikasi dengan </span><span class="font2" style="font-style:italic;">server</span><span class="font2">, tanpa harus me-</span><span class="font2" style="font-style:italic;">refresh</span><span class="font2"> semua halaman. Hal ini dimungkinkan karena </span><span class="font2" style="font-style:italic;">browser</span><span class="font2"> melakukan </span><span class="font2" style="font-style:italic;">request</span><span class="font2"> di belakang layar secara </span><span class="font2" style="font-style:italic;">asynchronous</span><span class="font2">. Gambar 2 mengilustrasikan interaksi antara </span><span class="font2" style="font-style:italic;">client </span><span class="font2">dan </span><span class="font2" style="font-style:italic;">server</span><span class="font2"> pada aplikasi </span><span class="font2" style="font-style:italic;">web </span><span class="font2">menggunakan Ajax.</span></p>
<p><span class="font2">Dengan bekerja sebagai lapisan tambahan diantara </span><span class="font2" style="font-style:italic;">browser</span><span class="font2"> pengguna dan </span><span class="font2" style="font-style:italic;">web server</span><span class="font2">, Ajax menangani komunikasi dengan </span><span class="font2" style="font-style:italic;">server</span><span class="font2"> di belakang layar, melakukan </span><span class="font2" style="font-style:italic;">request</span><span class="font2"> dan memproses data kembalian. Hasilnya diintegrasikan ke halaman yang sedang ditampilkan tanpa memerlukan </span><span class="font2" style="font-style:italic;">refresh </span><span class="font2">ata</span><span class="font2" style="text-decoration:underline;">u me-</span><span class="font2" style="font-style:italic;text-decoration:underline;">load</span><span class="font2" style="text-decoration:underline;"> suatu halaman yang bar</span><span class="font2">u.</span></p><img src="https://jurnal.harianregional.com/media/2690-2.jpg" alt="" style="width:163pt;height:119pt;">
<p><span class="font2">Gambar 2 Interaksi </span><span class="font2" style="font-style:italic;">client</span><span class="font2"> dan </span><span class="font2" style="font-style:italic;">server </span><span class="font2">pada aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> menggunakan Ajax Namun, Ajax juga memiliki kekurangan. Salah satu kekurangan yang paling utama adalah Ajax berbasis JavaScript yang sulit untuk ditulis, memakan banyak waktu untuk dikembangkan serta sangat sulit untuk di-</span><span class="font2" style="font-style:italic;">debug</span><span class="font2"> [MOR-06]. Dengan kata lain, untuk membuat suatu aplikasi </span><span class="font2" style="font-style:italic;">web</span></p>
<p><span class="font2">berbasis Ajax, pengembang harus menguasai JavaScript secara mendalam. Untuk &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;mengatasi &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;masalah</span></p>
<p><span class="font2">pengembangan aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> berbasis Ajax, banyak perusahaan yang membuat kode pustaka dan tool yang dapat dipakai oleh pengembang untuk membuat situs yang memasukkan teknik Ajax. Salah satu perusahaan tersebut adalah Microsoft yang mengembangkan suatu </span><span class="font2" style="font-style:italic;">framework</span><span class="font2"> yang diberi nama Atlas [SMI-06]. Atlas dimaksudkan agar para pengembang ASP.NET bisa membuat aplikasi </span><span class="font2" style="font-style:italic;">web </span><span class="font2">yang berbasis Ajax dengan mudah dan cepat, tanpa harus menjadi seorang pemrogram ahli JavaScript.</span></p>
<p><span class="font2">Dengan memanfaatkan keunggulan teknik Ajax yang dikombinasikan dengan kemudahan pengembangan aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> dengan menggunakan ASP.NET dan ASP.NET Ajax, penulis mencoba untuk membuat sebuah aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> layanan informasi pariwisata Jogja. Aplikasi ini berisi tentang informasi-informasi mengenai tempat-tempat maupun </span><span class="font2" style="font-style:italic;">event-event </span><span class="font2">wisata di Daerah Istimewa Yogyakarta. Tujuan penelitian ini adalah ingin mengetahui apakah aplikasi </span><span class="font2" style="font-style:italic;">web</span><span class="font2"> Ajax memiliki waktu </span><span class="font2" style="font-style:italic;">loading</span><span class="font2"> lebih singkat dibandingkan dengan aplikasi </span><span class="font2" style="font-style:italic;">web </span><span class="font2">tradisional. Aplikasi web Ajax dibangun dengan menggunakan ASP.NET dan ASP.NET Ajax, sedangkan aplikasi web tradisional dibangun dengan hanya menggunakan ASP.NET.</span></p>
<h1><a name="bookmark8"></a><span class="font2" style="font-weight:bold;"><a name="bookmark9"></a>METODE PENELITIAN</span></h1>
<p><span class="font2">Penelitian ini menggunakan metode teoritik dan aplikasi, dengan cara mempelajari karya-karya ilmiah yang telah dihimpun dan diimplementasikan dengan membuat aplikasi web layanan informasi pariwisata Jogja, baik menggunakan konsep tradisional maupun menggunakan konsep Ajax dengan ASP.NET Ajax.</span></p>
<p><span class="font2">Pengujian yang dilakukan pada penelitian ini menggunakan metode statistik analisis variansi dua arah, untuk mengetahui apakah aplikasi web Ajax dengan menggunakan Atlas lebih cepat dibanding aplikasi web tradisional.</span></p>

<h1><a name="bookmark12"></a><span class="font2" style="font-weight:bold;"><a name="bookmark13"></a>KESIMPULAN</span></h1>
<p><span class="font2">Aplikasi web Ajax dengan ASP.NET Ajax memiliki waktu loading yang lebih lambat dibandingkan aplikasi web tradisional pada saat dilakukan pengujian loading satu halaman penuh. Hal ini disebabkan ASP.NET Ajax melewatkan jumlah data yang lebih besar karena adanya penambahan berkas javascript sebesar 20.931 byte. Namun, waktu loading aplikasi web Ajax dengan ASP.NET Ajax pada saat menampilkan bagian-bagian tertentu pada suatu halaman lebih cepat dibandingkan aplikasi web tradisional. Hal ini disebabkan karena ASP.NET Ajax hanya meng-</span><span class="font2" style="font-style:italic;">update</span><span class="font2"> bagian-bagian tertentu saja, sehingga melewatkan data yang lebih sedikit dibanding.</span></p>
