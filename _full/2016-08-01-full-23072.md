<p><span class="font1" style="font-weight:bold;font-style:italic;">Abstrak</span></p>
<p><span class="font1" style="font-style:italic;">Tugas pengelolaan jaringan yang dilakukan administrator jaringan diantaranya yaitu pengumpulan informasi resource jaringan yang tersedia. Teknologi SNMP (Simple Network Management Protocol) memberikan fleksibilitas bagi administrator jaringan dalam mengatur network secara keseluruhan dari satu lokasi. Aplikasi Network Monitoring Tools berbasis Agent JAVA terdiri dari Master agent yang bertugas untuk melakukan management Request agent serta akses database. Request agent yang bertugas untuk melakukan pemantauan server yang mengimplementasi library SNMP4j dengan sistem multi-agent. Disisi interface, aplikasi Network Monitoring Tools menggunakan media web sebagai interface administrator sehingga dapat digunakan darimana saja dan kapan saja. Hasil dari penelitian ini memperlihatkan bahwa aplikasi yang dibuat bekerja sebagai Network Monitoring Tools mampu bekerja dengan persen error pada kisaran 0-18%. Selain itu Aplikasi ini menghasilkan tren pembacaan data server lebih stabil dan cepat dibandingkan dengan aplikasi Cacti. Hal ini didukung oleh kemampuan Request Agent yang mampu merespon tingkat beban kerja server yang di pantau.</span></p>
<p><span class="font1" style="font-weight:bold;font-style:italic;">Kata kunci </span><span class="font1" style="font-style:italic;">: Agent, JAVA, SNMP, Cacti, Network monitoring.</span></p>
<p><span class="font1" style="font-weight:bold;font-style:italic;">Abstract</span></p>
<p><span class="font1" style="font-style:italic;">Network management tasks are performed by the Network Administrator such as gathering available network resources information. The SNMP (Simple Network Management Protocol) Technology provides flexibility for network administrators in managing the overall network from a single location. Agent based Network Monitoring Tools JAVA Application consists of Master agent whose job is to perform as well as the management agent Request and database access. The Request agent in charge for monitoring servers that implement SNMP4j library with multi -agent systems. For interface, Network Monitoring Tools application using web media as an administrator interface that can be used from anywhere and at anytime. The results of this study showed that the application as a Network Monitoring Tools are able to work with the percent error in the range of 0-18 % . Besides these applications generate trend data readout server more stable and faster than the application Cacti do. This is supported by the ability of the Request Agent to respond the level of server workloads</span></p>
<p><span class="font1" style="font-weight:bold;font-style:italic;">Keywords </span><span class="font1" style="font-style:italic;">: Agent, JAVA, SNMP, Cacti, Network monitoring.</span></p>

<h2><a name="bookmark2"></a><span class="font1" style="font-weight:bold;"><a name="bookmark3"></a>1. &nbsp;&nbsp;&nbsp;Pendahuluan</span></h2></li></ul>
<p><span class="font1">Tugas pengelolaan jaringan yang dilakukan </span><span class="font1" style="font-style:italic;">administrator</span><span class="font1"> jaringan memiliki sejumlah kesulitan, diantaranya yaitu pengumpulan informasi </span><span class="font1" style="font-style:italic;">resource</span><span class="font1"> jaringan yang tersedia serta melakukan pemantauan terhadap </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> dan </span><span class="font1" style="font-style:italic;">router</span><span class="font1"> yang beroperasi pada jaringan. Manajemen sistem jaringan komputer berbasiskan pada teknologi SNMP (</span><span class="font1" style="font-style:italic;">Simple Network Management Protocol</span><span class="font1">) yang memberikan fleksibilitas bagi </span><span class="font1" style="font-style:italic;">administrator</span><span class="font1"> jaringan dalam mengatur </span><span class="font1" style="font-style:italic;">network</span><span class="font1"> secara keseluruhan dari satu lokasi [1]. Banyak aplikasi 3<sup>rd</sup> </span><span class="font1" style="font-style:italic;">party</span><span class="font1"> seperti cacti, MRTG, monit, munin,</span></p>
<p><span class="font1">nagios, zenos dan zabix yang menyediakan banyak fitur dalam manajemen sistem jaringan komputer menggunakan protokol SNM, dan diakses melalui web browser [2]. Namun aplikasi-aplikasi tersebut cukup sulit dikustomisasi menyesuaikan dengan kebutuhan instansi pengguna. Dibutuhkan sebuah aplikasi mandiri yang sederhana agar mudah disesuaikan dengan kebutuhan dan memiliki fitur-fitur </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1"> seperti pada aplikasi 3<sup>rd</sup> </span><span class="font1" style="font-style:italic;">party</span><span class="font1"> pada umumnya. Untuk menjawab permasalahan tersebut maka dibuatlah sebuah aplikasi </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span></p>
<p><span class="font1">(NMT) berbasis </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> JAVA. &nbsp;</span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> JAVA &nbsp;mampu bekerja secara berulang-ulang</span></p>
<p><span class="font1">(</span><span class="font1" style="font-style:italic;">Autonomous</span><span class="font1">) dan merespon sesuai dengan parameter yang diterima. Penggunaan </span><span class="font1" style="font-style:italic;">Agent </span><span class="font1">JAVA sebagai pemantau </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> yang mampu merespon kondisi pembebanan </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> dalam bentuk interval pengambilan datanya (</span><span class="font1" style="font-style:italic;">request scheduling</span><span class="font1">) diharapkan dapat menjadi solusi untuk aplikasi mandiri </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> pada jaringan Fakultas Teknik Universitas Mataram.</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark4"></a><span class="font1" style="font-weight:bold;"><a name="bookmark5"></a>2. &nbsp;&nbsp;&nbsp;Landasan Teori</span><br><br><span class="font1" style="font-weight:bold;"><a name="bookmark6"></a>2.1. &nbsp;&nbsp;&nbsp;SNMP</span></h2></li></ul>
<p><span class="font1">Secara sederhana, SNMP merupakan sebuah protokol yang didesain untuk memberikan kemampuan kepada pemakai untuk mengelola jaringan komputernya dari jarah jauh atau remote[3]. Adanya SNMP memungkinkan manajemen jaringan yang tersentralisasi, kuat, dan kompatibel pada semua platform. Tujuan utama dari protokol SNMP hanya pada satu tujuan yaitu melakukan </span><span class="font1" style="font-style:italic;">remote</span><span class="font1"> manajemen dari peralatan [4]. Sebuah jaringan yang dapat di-</span><span class="font1" style="font-style:italic;">manage </span><span class="font1">menggunakan SNMP pada dasarnya memiliki tiga komponen, yaitu </span><span class="font1" style="font-style:italic;">Managed Device, Agent SNMP,</span><span class="font1"> dan </span><span class="font1" style="font-style:italic;">Network</span><span class="font1">-</span><span class="font1" style="font-style:italic;">management System</span><span class="font1"> [4]. Sebuah </span><span class="font1" style="font-style:italic;">managed device</span><span class="font1"> adalah sebuah </span><span class="font1" style="font-style:italic;">node </span><span class="font1">(dapat berupa </span><span class="font1" style="font-style:italic;">server, switch, router</span><span class="font1">) di jaringan yang berisi </span><span class="font1" style="font-style:italic;">Agent SNMP</span><span class="font1"> yang berada di jaringan yang dapat di </span><span class="font1" style="font-style:italic;">manage</span><span class="font1">. </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> SNMP adalah sebuah modul </span><span class="font1" style="font-style:italic;">software network management</span><span class="font1"> yang berada di dalam </span><span class="font1" style="font-style:italic;">managed device</span><span class="font1">. </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> ini mengetahui tentang informasi manajemen dan dalam menterjemahkan ke informasi yang kompatibel dengan SNMP. Pada sistem operasi linux telah disediakan aplikasi snmpd sebagai </span><span class="font1" style="font-style:italic;">agent</span><span class="font1"> SNMP. Aplikasi NMS menjalankan aplikasi yang dapat memonitor dan mengontrol </span><span class="font1" style="font-style:italic;">managed device</span><span class="font1">. NMS memberikan </span><span class="font1" style="font-style:italic;">resource memory</span><span class="font1"> dan </span><span class="font1" style="font-style:italic;">prosesor</span><span class="font1"> yang dibutuhkan untuk manajemen </span><span class="font1" style="font-style:italic;">network</span><span class="font1">.</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark7"></a><span class="font1" style="font-weight:bold;"><a name="bookmark8"></a>2.2. &nbsp;&nbsp;&nbsp;Agent Java</span></h2></li></ul>
<p><span class="font1" style="font-style:italic;">Software Agent</span><span class="font1"> (selanjutnya di sebut </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> saja) adalah entitas perangkat lunak yang didedikasikan untuk tujuan tertentu. </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> bisa memiliki ide sendiri mengenai bagaimana menyelesaikan suatu pekerjaan tertentu atau agenda tersendiri. Karakteristik dari </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> [5]:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">a.</span><span class="font1" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;Autonomy</span><span class="font1">: komputer umumnya hanya berespon pada manipulasi langsung. Kontras dengan </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> perangkat lunak yang mengamati lingkungannya dan bisa melakukan tindakan otonom.</span></p></li>
<li>
<p><span class="font1">b. &nbsp;</span><span class="font1" style="font-style:italic;">Reaktif</span><span class="font1">: suatu software Agent berespon dalam waktu yang bermacam-macam untuk</span></p></li></ul>
<p><span class="font1">merubah dalam lingkungannya</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">c. &nbsp;</span><span class="font1" style="font-style:italic;">Goal driven</span><span class="font1">: suatu Agent bisa menerima </span><span class="font1" style="font-style:italic;">Request</span><span class="font1"> tingkat tinggi yang menentukan</span></p></li></ul>
<p><span class="font1">tujuan dari </span><span class="font1" style="font-style:italic;">user</span><span class="font1"> manusia (atau </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> lainnya) dan memutuskan dimana dan bagaimana untuk menjawab </span><span class="font1" style="font-style:italic;">Request</span><span class="font1"> tersebut.</span></p>
<p><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> yang tidak berpindah ke </span><span class="font1" style="font-style:italic;">host</span><span class="font1"> lain disebut </span><span class="font1" style="font-style:italic;">stationary Agent</span><span class="font1">. </span><span class="font1" style="font-style:italic;">Multi agent system</span><span class="font1"> adalah sebuah </span><span class="font1" style="font-style:italic;">system</span><span class="font1"> yang memungkinkan sejumlah </span><span class="font1" style="font-style:italic;">agent</span><span class="font1"> (</span><span class="font1" style="font-style:italic;">multi agent</span><span class="font1">) bekerja sama untuk menyelesaikan suatu masalah yang tidak dapat dikerjakan oleh individu </span><span class="font1" style="font-style:italic;">agent</span><span class="font1"> [5]. Pada </span><span class="font1" style="font-style:italic;">multi agent system</span><span class="font1">, setiap </span><span class="font1" style="font-style:italic;">agent</span><span class="font1"> bekerja secara </span><span class="font1" style="font-style:italic;">multithreading</span><span class="font1"> diatas </span><span class="font1" style="font-style:italic;">runtime</span><span class="font1"> JAVA sehingga memberikan impresi bahwa </span><span class="font1" style="font-style:italic;">agent</span><span class="font1"> mampu bekerja secara </span><span class="font1" style="font-style:italic;">paralel independent</span><span class="font1">.</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark9"></a><span class="font1" style="font-weight:bold;"><a name="bookmark10"></a>2.3. &nbsp;&nbsp;&nbsp;JADE</span></h2></li></ul>
<p><span class="font1">JADE (</span><span class="font1" style="font-style:italic;">Java Agent DEvelopment Framework</span><span class="font1">) merupakan sebuah kerangka kerja perangkat lunak yang diimplementasikan sepenuhnya dalam bahasa </span><span class="font1" style="font-style:italic;">Java</span><span class="font1"> [6]. JADE memudahkan implementasi dari </span><span class="font1" style="font-style:italic;">Multi-Agent sistem</span><span class="font1"> (MAS) melalui </span><span class="font1" style="font-style:italic;">middleware</span><span class="font1"> yang bekerja sesuai spesifikasi FIPA (</span><span class="font1" style="font-style:italic;">Foundation for Intelligent Physical Agent</span><span class="font1">). Karena sepenuhnya diimplementasikan dalam bahasa pemrograman </span><span class="font1" style="font-style:italic;">Java</span><span class="font1">, maka JADE juga mendapatkan seluruh keuntungan dari bahasa pemrograman tersebut, termasuk ketidak-tergantungan pada arsitektur </span><span class="font1" style="font-style:italic;">platform</span><span class="font1">. </span><span class="font1" style="font-style:italic;">Agent platform</span></p>
<p><span class="font1">pada JADE dapat didistribusikan di beberapa mesin yang berbeda, dan tidak perlu</span></p>
<p><span class="font1">menggunakan sistem operasi yang sama.</span></p>

<h2><a name="bookmark11"></a><span class="font1" style="font-weight:bold;"><a name="bookmark12"></a>3. &nbsp;&nbsp;&nbsp;Metode Penelitian</span><br><br><span class="font1" style="font-weight:bold;"><a name="bookmark13"></a>3.1. &nbsp;&nbsp;&nbsp;Analisis Kebutuhan</span></h2></li></ul>
<p><span class="font1">Sistem </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1"> yang saat ini berjalan banyak menggunakan </span><span class="font1" style="font-style:italic;">tools</span><span class="font1">-</span><span class="font1" style="font-style:italic;">tools</span><span class="font1"> buatan luar negeri yang memanfaatkan protokol SNMP sebagai media </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1"> pada suatu </span><span class="font1" style="font-style:italic;">server</span><span class="font1">. Aplikasi yang baru tetap menggunakan protokol SNMP sebagai media </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1"> agar tidak terjadi perubahan langsung pada konfigurasi </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> dengan adanya penggantian aplikasi </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1">. </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> yang akan dibangun adalah pemanfaatan dari teknologi </span><span class="font1" style="font-style:italic;">Agent JAVA, </span><span class="font1">yang mana untuk proses </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1"> dalam satu </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> ditangani oleh sebuah </span><span class="font1" style="font-style:italic;">Agent.</span><span class="font1"> Semakin banyak </span><span class="font1" style="font-style:italic;">host</span><span class="font1"> yang perlu dipantau maka </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> yang diperlukan juga akan semakin banyak dan membutuhkan spesifikasi komputer yang cukup tinggi. </span><span class="font1" style="font-style:italic;">Interval Request Data</span><span class="font1"> akan diatur oleh </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> dengan memantau tingkatan </span><span class="font1" style="font-style:italic;">traffic</span><span class="font1"> jaringan dan beban kerja </span><span class="font1" style="font-style:italic;">host</span><span class="font1">. Ini artinya </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> yang akan melakukan </span><span class="font1" style="font-style:italic;">Request</span><span class="font1"> data akan secara bergiliran melakukan pengambilan data, sehingga </span><span class="font1" style="font-style:italic;">traffic</span><span class="font1"> jaringan lebih stabil tanpa mengurangi performa aplikasi </span><span class="font1" style="font-style:italic;">monitoring</span><span class="font1">.</span></p>

<h2><a name="bookmark14"></a><span class="font1" style="font-weight:bold;"><a name="bookmark15"></a>3.2. &nbsp;&nbsp;&nbsp;Perancangan Sistem</span></h2></li></ul>
<p><span class="font1">Perangkat lunak yang akan dibangun adalah sebuah </span><span class="font1" style="font-style:italic;">tools</span><span class="font1"> untuk memantau jaringan atau </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1">. Pemantauan jaringan dilakukan dengan cara memanfaatkan protokol SNMP untuk mengakses MIB (</span><span class="font1" style="font-style:italic;">Management Information Bases</span><span class="font1">) tiap </span><span class="font1" style="font-style:italic;">host</span><span class="font1"> dan mendapatkan data sesuai OID (</span><span class="font1" style="font-style:italic;">Object ID</span><span class="font1">) yang dikirimkan oleh aplikasi </span><span class="font1" style="font-style:italic;">daemon</span><span class="font1"> snmpd. Proses permintaan data </span><span class="font1" style="font-style:italic;">resource</span><span class="font1"> menggunakan </span><span class="font1" style="font-style:italic;">library</span><span class="font1"> JAVA SNMP4j dan menghasilkan </span><span class="font1" style="font-style:italic;">data set</span><span class="font1"> dari </span><span class="font1" style="font-style:italic;">resource server</span><span class="font1"> yang dipantau. </span><span class="font1" style="font-style:italic;">Data set</span><span class="font1"> ini kemudian di simpan kedalam </span><span class="font1" style="font-style:italic;">database</span><span class="font1"> dan ditampilkan pada aplikasi </span><span class="font1" style="font-style:italic;">web</span><span class="font1"> yang </span><span class="font1" style="font-style:italic;">di</span><span class="font1"> akses oleh </span><span class="font1" style="font-style:italic;">client</span><span class="font1"> secara </span><span class="font1" style="font-style:italic;">realtime</span><span class="font1"> sesuai data yang diterima.</span></p><img src="https://jurnal.harianregional.com/media/23072-1.jpg" alt="" style="width:424pt;height:298pt;">
<p><span class="font1" style="font-weight:bold;">Gambar 1. </span><span class="font1">Ilustrasi Sistem Monitoring</span></p>
<p><span class="font1">Berikut penjelasan alur proses monitoring jaringan :</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">a. &nbsp;Dengan menggunakan </span><span class="font1" style="font-style:italic;">interface web</span><span class="font1">, </span><span class="font1" style="font-style:italic;">admin</span><span class="font1"> mendaftarkan semua </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> yang ingin</span></p></li></ul>
<p><span class="font1">dipantau kedalam Sistem</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">b. &nbsp;Perangkat lunak yang akan dibangun adalah sebuah tools untuk memantau jaringan</span></p></li></ul>
<p><span class="font1">atau &nbsp;</span><span class="font1" style="font-style:italic;">Network Monitoring &nbsp;Tools</span><span class="font1">. &nbsp;Pemantauan jaringan &nbsp;dilakukan &nbsp;dengan cara</span></p>
<p><span class="font1">memanfaatkan protokol SNMP untuk mengakses MIB (</span><span class="font1" style="font-style:italic;">Management Information Bases</span><span class="font1">) dan disimpan pada </span><span class="font1" style="font-style:italic;">database komputer monitoring</span><span class="font1">. Pendaftaran terdiri dari IP </span><span class="font1" style="font-style:italic;">address</span><span class="font1">, </span><span class="font1" style="font-style:italic;">port</span><span class="font1"> SNMP, dan </span><span class="font1" style="font-style:italic;">community Key</span><span class="font1"> untuk mengakses snmpd pada </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> tersebut.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">c. &nbsp;&nbsp;&nbsp;Aplikasi mencetak 2 jenis </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> yaitu </span><span class="font1" style="font-style:italic;">Master Agent</span><span class="font1"> dan </span><span class="font1" style="font-style:italic;">Request Agent</span><span class="font1">. </span><span class="font1" style="font-style:italic;">Master Agent </span><span class="font1">adalah </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> yang bertugas untuk mencetak </span><span class="font1" style="font-style:italic;">Request Agent</span><span class="font1"> sesuai dengan </span><span class="font1" style="font-style:italic;">data host </span><span class="font1">yang terdapat pada </span><span class="font1" style="font-style:italic;">database</span><span class="font1">, menerima data </span><span class="font1" style="font-style:italic;">resource</span><span class="font1"> jaringan dari </span><span class="font1" style="font-style:italic;">Request Agent</span><span class="font1"> dan meng-</span><span class="font1" style="font-style:italic;">entry</span><span class="font1"> data tersebut ke </span><span class="font1" style="font-style:italic;">database</span><span class="font1"> aplikasi. </span><span class="font1" style="font-style:italic;">Request Agent</span><span class="font1"> adalah </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> yang akan melakukan </span><span class="font1" style="font-style:italic;">Request data</span><span class="font1"> ke </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> yang ingin dipantau</span><span class="font1" style="font-style:italic;">,</span><span class="font1"> melakukan pengolahan data mengirimkan hasilnya ke </span><span class="font1" style="font-style:italic;">Master Agent</span><span class="font1">.</span></p></li>
<li>
<p><span class="font3">d.</span><span class="font1" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;Request Agent</span><span class="font1"> akan melakukan </span><span class="font1" style="font-style:italic;">Request resource data</span><span class="font1"> yang terdiri dari </span><span class="font1" style="font-style:italic;">bandwidth traffic</span><span class="font1">, </span><span class="font1" style="font-style:italic;">memory usage</span><span class="font1">, CPU </span><span class="font1" style="font-style:italic;">usage</span><span class="font1">, dan </span><span class="font1" style="font-style:italic;">disk usage</span><span class="font1"> ke </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> secara bergantian dengan interval waktu berdasarkan tingkat beban kerja dan </span><span class="font1" style="font-style:italic;">traffic server</span><span class="font1">. Interval waktu ini</span></p></li></ul>
<p><span class="font1">diperoleh dari kategori tingkat </span><span class="font1" style="font-style:italic;">traffic</span><span class="font1"> data dan penggunaan CPU serta RAM.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">e.</span><span class="font1"> &nbsp;&nbsp;&nbsp;Pada </span><span class="font1" style="font-style:italic;">Web Application</span><span class="font1">, sistem akan mengakses </span><span class="font1" style="font-style:italic;">database</span><span class="font1"> secara </span><span class="font1" style="font-style:italic;">realtime</span><span class="font1"> untuk</span></p></li></ul>
<p><span class="font1">mendapatkan informasi </span><span class="font1" style="font-style:italic;">resource</span><span class="font1"> jaringan yang telah dikumpulkan oleh </span><span class="font1" style="font-style:italic;">Master Agent</span><span class="font1">.</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark16"></a><span class="font1" style="font-weight:bold;"><a name="bookmark17"></a>3.3. &nbsp;&nbsp;&nbsp;Programming</span></h2></li></ul>
<p><span class="font1">Berikut beberapa prosedur yang digunakan dalam pembuatan program </span><span class="font1" style="font-style:italic;">Network Monitoring </span><span class="font1">JAVA :</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1" style="font-style:italic;">a.</span><span class="font1"> &nbsp;&nbsp;&nbsp;Prosedur utilitas sistem</span></p></li></ul>
<p><span class="font1">Prosedur ini berfungsi sebagai pendukung aplikasi </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1">. Terdiri dari koneksi </span><span class="font1" style="font-style:italic;">database</span><span class="font1">, akses file konfigurasi dan penyedia </span><span class="font1" style="font-style:italic;">logging error</span><span class="font1">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1" style="font-style:italic;">b.</span><span class="font1"> &nbsp;&nbsp;&nbsp;Prosedur </span><span class="font1" style="font-style:italic;">Agent</span></p></li></ul>
<p><span class="font1">Prosedur ini berfungsi sebagai prosedur utama </span><span class="font1" style="font-style:italic;">agent</span><span class="font1">. Pada prosedur </span><span class="font1" style="font-style:italic;">agent</span><span class="font1"> terdiri dari</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">2 </span><span class="font1" style="font-style:italic;">class</span><span class="font1"> yaitu </span><span class="font1" style="font-style:italic;">class master agent</span><span class="font1"> dan </span><span class="font1" style="font-style:italic;">class request agent.</span></p></li></ul>
<ul style="list-style:none;"><li>
<p><span class="font1">c. &nbsp;&nbsp;&nbsp;Prosedur komunikasi </span><span class="font1" style="font-style:italic;">Agent</span></p></li></ul>
<p><span class="font1">Prosedur ini digunakan oleh </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> untuk berkomunikasi dan bertukar data antar </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1" style="font-style:italic;">d.</span><span class="font1"> &nbsp;&nbsp;&nbsp;Prosedur pemantauan </span><span class="font1" style="font-style:italic;">server</span></p></li></ul>
<p><span class="font1">Prosedur ini digunakan oleh </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> untuk melakukan pemantauan </span><span class="font1" style="font-style:italic;">server.</span><span class="font1"> Tediri dari pembentukan koneksi </span><span class="font1" style="font-style:italic;">protocol</span><span class="font1"> UDP, pengaksesan </span><span class="font1" style="font-style:italic;">protocol</span><span class="font1"> SNMP dan proses-proses pengolahan data yang diterima oleh </span><span class="font1" style="font-style:italic;">Agent</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark18"></a><span class="font1" style="font-weight:bold;"><a name="bookmark19"></a>3.4. &nbsp;&nbsp;&nbsp;Pengujian</span></h2></li></ul>
<p><span class="font1">Dalam pembuatan sistem, pengujian ditujukan untuk mengetahui kinerja sistem dalam melakukan proses pengumpulan data </span><span class="font1" style="font-style:italic;">resource server</span><span class="font1">. Tahapan pengujian yang dilakukan antara lain :</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">a. &nbsp;&nbsp;&nbsp;Pengujian pembacaan data protokol SNMP</span></p></li></ul>
<p><span class="font1">Protokol SNMP diakses dengan memanfaatkan </span><span class="font1" style="font-style:italic;">library</span><span class="font1"> tambahan SNMP4j dengan </span><span class="font1" style="font-style:italic;">output</span><span class="font1"> berupa sebuah </span><span class="font1" style="font-style:italic;">set data</span><span class="font1"> yang siap disimpan ke </span><span class="font1" style="font-style:italic;">database.</span><span class="font1"> Pengujian dilakukan dengan memeriksa apakah </span><span class="font1" style="font-style:italic;">library</span><span class="font1"> tersebut dapat memberikan respon </span><span class="font1" style="font-style:italic;">output set data </span><span class="font1">sesuai </span><span class="font1" style="font-style:italic;">input</span><span class="font1"> OIDnya.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font1">b. &nbsp;&nbsp;&nbsp;Pengujian perbandingan hasil pemantauan dengan aplikasi Cacti</span></p></li></ul>
<p><span class="font1">Pengujian perbandingan hasil pemantauan dengan aplikasi Cacti berfungsi untuk mengetahui kinerja </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> dibandingkan dengan kinerja cacti sebagai aplikasi yang telah banyak digunakan.</span></p>

<h2><a name="bookmark23"></a><span class="font1" style="font-weight:bold;"><a name="bookmark24"></a>4.2. &nbsp;&nbsp;&nbsp;Pengujian perbandingan hasil pemantauan dengan aplikasi Cacti</span></h2></li></ul>
<p><span class="font1">Pada pengujian ini hasil pemantauan aplikasi WLTI </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> akan dibandingkan dengan hasil pemantauan aplikasi Cacti. Pengujian ini bertujuan untuk mengetahui perbandingan kinerja aplikasi WLTI </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> dengan aplikasi yang telah digunakan secara umum. Berikut perbandingan data hasil pemantauan </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> yang diambil secara bersamaan dengan kondisi </span><span class="font1" style="font-style:italic;">server idle</span><span class="font1"> :</span></p><img src="https://jurnal.harianregional.com/media/23072-2.jpg" alt="" style="width:417pt;height:194pt;">
<p><span class="font1" style="font-weight:bold;">Gambar 2. </span><span class="font1">Grafik data hasil pemantauan RAM dan CPU server oleh cacti</span></p><img src="https://jurnal.harianregional.com/media/23072-3.jpg" alt="" style="width:418pt;height:295pt;">
<p><span class="font1" style="font-weight:bold;">Gambar 3. </span><span class="font1">Grafik data hasil pemantauan RAM dan CPU server oleh WLTI NMT</span></p>
<p><span class="font1">Dari kedua perbandingan diatas keanehan muncul pada data hasil pemantauan cacti. Secara logis kondisi </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> yang </span><span class="font1" style="font-style:italic;">idle</span><span class="font1"> memiliki besaran penggunaan RAM dan CPU berada pada </span><span class="font1" style="font-style:italic;">level </span><span class="font1">terendah dan stabil (tidak berubah-ubah). Sedangkan pada grafik hasil pemantauan cacti (gambar 2) dapat dilihat terdapat fluktuasi penggunaan </span><span class="font1" style="font-style:italic;">resource server</span><span class="font1"> yang terus meningkat. Sedangkan pada aplikasi WLTI penggunaan RAM dan CPU stabil di</span><span class="font1" style="font-style:italic;">level</span><span class="font1"> 9% dan 0%. Hal ini logis mengingat kondisi </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> yang </span><span class="font1" style="font-style:italic;">idle</span><span class="font1"> dan dapat dicocokkan dengan aplikasi </span><span class="font1" style="font-style:italic;">monitoring </span><span class="font1">bawaan linux seperti </span><span class="font1" style="font-style:italic;">top</span><span class="font1">. Diluar dari kompleksitas, fitur-fitur dan cakupan entitas jaringan yang mampu dipantau oleh cacti, dari kedua grafik perbandingan diatas dapat disimpulkan bahwa aplikasi WLTI NMT memiliki akurasi pemantauan yang lebih baik dari cacti. Selain itu aplikasi</span></p>
<p><span class="font1">WLTI NMT mampu melakukan pengambilan data dengan interval hingga 20 detik sedangkan cacti hanya menyediakan interval di 5 menit.</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark25"></a><span class="font1" style="font-weight:bold;"><a name="bookmark26"></a>5. &nbsp;&nbsp;&nbsp;Kesimpulan</span></h2></li></ul>
<p><span class="font1">Dari penelitian yang telah dilakukan, dapat ditarik kesimpulan sebagai berikut ; Persentase </span><span class="font1" style="font-style:italic;">error </span><span class="font1">pembacaan data oleh aplikasi sebesar 3.74% untuk CPU </span><span class="font1" style="font-style:italic;">Usage,</span><span class="font1"> 11.57% untuk RAM </span><span class="font1" style="font-style:italic;">Usage</span><span class="font1">, 17.64% untuk </span><span class="font1" style="font-style:italic;">Traffic</span><span class="font1"> IN dan 10.63% untuk </span><span class="font1" style="font-style:italic;">Traffic</span><span class="font1"> Out. Hal ini dipengaruhi oleh </span><span class="font1" style="font-style:italic;">interval </span><span class="font1">pengambilan data dan sistem pembulatan besaran data. Sedangkan persentase </span><span class="font1" style="font-style:italic;">error </span><span class="font1">pembacaan data untuk </span><span class="font1" style="font-style:italic;">disk usage</span><span class="font1"> dan </span><span class="font1" style="font-style:italic;">disk free</span><span class="font1"> sebesar 0%. Hasil pengujian perbandingan antara WLTI NMT dengan Cacti menunjukkan bahwa pada kondisi </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> istirahat </span><span class="font1" style="font-style:italic;">(idle)</span><span class="font1"> data hasil pembacaan oleh Cacti menampilkan grafik penggunaan RAM yang cenderung naik dan grafik penggunaan CPU yang konstan. Sedangkan data hasil pembacaan oleh WLTI NMT menampilkan grafik penggunaan RAM dan CPU yang konstan. Hal ini berarti aplikasi WLTI </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> lebih akurat dibanding Cacti. Pengambilan data oleh aplikasi </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> yang mampu bekerja dengan interval maksimum sebesar 20 detik sekali, sedangkan Cacti hanya mampu bekerja dengan interval sebesar 5 menit sekali. Hal ini berarti bahwa WLTI </span><span class="font1" style="font-style:italic;">Network Monitoring Tools</span><span class="font1"> mampu menyajikan data lebih akurat. </span><span class="font1" style="font-style:italic;">Agent</span><span class="font1"> JAVA berbasis JADE dapat diimplementasikan sebagai pemantau </span><span class="font1" style="font-style:italic;">server</span><span class="font1"> dengan memanfaatkan </span><span class="font1" style="font-style:italic;">library SNMP4j</span><span class="font1"> sebagai penyedia layanan akses ke protokol SNMP.</span></p>
