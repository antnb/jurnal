<p><span class="font2" style="font-weight:bold;font-style:italic;">Abstrak</span></p>
<p><span class="font2" style="font-style:italic;">Rivest code 4 (RC4) merupakan algoritma kriptografi modern kunci simetris yang mempunyai mode operasi stream chipper, sehingga dalam memproses data dan informasi pada saat tertentu serta menggunakan dua buah substitution box (s-box) berupa array dengan panjang permutasi 256 dan s-box kedua berupa permutasi yang merupakan fungsi dari kunci publik. Algoritma rivest code 4 digunakan untuk mengekripsi data, pesan atau informasi. Sistem informasi inventory stock barang pada distributor PT. Wings Food merupakan sistem yang memproses data barang masuk dan barang keluar, sehingga data yang tersimpan didalam database harus terjamin kerahasiaan dan keamanannya</span><span class="font9" style="font-weight:bold;font-style:italic;">. </span><span class="font2" style="font-style:italic;">Untuk menjaga keamanan data stock barang yang di simpan di dalam database, maka di kombinasikanlah dengan algoritma rivest code 4. Dengan demikian, data yang terenkripsi pada database tidak akan diketahui arti yang sebenarnya dan sulit untuk dirubah ataupun diganti. Kunci simetris yang digunakan untuk proses enkripsi dan deskripsi algoritma rivest code 4 pada sistem informasi inventory stock barang, dimasukkan ke dalam source code. Implementasi algoritma rivest code 4 dibuat dengan bahasa pemrograman Hypertext Preprocessor dan database menggunakan MySQL. Berdasarkan hasil, data yang tersimpan dalam database sistem informasi inventory stock barang pada distributor PT. Wings Food terenkripsi sehingga data yang tersimpan di dalam database tersebut terjamin kerahasiaan dan keamanannya serta data yang ditampilkannya tidak terenkripsi.</span></p>
<p><span class="font2" style="font-weight:bold;font-style:italic;">Kata Kunci : </span><span class="font2" style="font-style:italic;">keamanan, algoritma rivest code 4, database.</span></p>
<h2><a name="bookmark2"></a><span class="font3" style="font-weight:bold;font-style:italic;"><a name="bookmark3"></a>Abstract</span></h2>
<p><span class="font2" style="font-style:italic;">Rivest code 4 (RC4) is a cryptographic algorithm modern symentic key which has an operating mode stream chipper. It’s used two substitutions box in processing data and information, they are array with a permutation length of 256 and second s-box as permutation with has a function as public key. Algorithm rivest code 4 was used for data encryption, message or information. Information system inventory stock of goods at distributor PT. Wings Food is a system that processed incoming and outgoing goods data, so that the data stored in database must be guaranteed confidential and its secure. To keep the security of stock data items stored in database, it was combined with algorithm rivest code 4. Thus, the data encrypted in database would not be known the real meaning and was difficult to change or replaced. Symmetry key that was used for processing encryption and description rivest code 4 algorithm at information system inventory stock of goods in source code. Algorithm implementation rivest code 4 was made by programming language Hypertext Preprocessor and MySQL. Based on the result, the data displayed in database PT. Wings Food was encrypted, so it was found that the system could keep the data confidential in a secure. In short, the data displayed was unencrypted.</span></p>

<h3><a name="bookmark8"></a><span class="font2" style="font-weight:bold;"><a name="bookmark9"></a>3. &nbsp;&nbsp;&nbsp;Kajian Pustaka</span><br><br><span class="font2" style="font-weight:bold;"><a name="bookmark10"></a>3.1. &nbsp;&nbsp;&nbsp;Pengertian Kriptografi</span></h3></li></ul>
<p><span class="font2">Kriptografi dikenal sebagai ilmu yang mempelajari bagaimana menyembunyikan pesan. Namun pada pengertian modern kriptografi adalah ilmu yang bersandarkan pada teknik matematika untuk berurusan dengan keamanan informasi seperti kerahasiaan, keutuhan data, dan otentikasi entitas. Jadi pengertian kriptografi modern adalah ilmu yang tidak mempelajari tentang penyembunyian pesan namun lebih pada sekumpulan teknik yang menyediakan keamanan informasi [5].</span></p>
<p><span class="font2">Kriptografi merupakan seni dan ilmu menyembunyikan informasi dari penerima yang tidak berhak </span><span class="font2" style="font-style:italic;">(unauthorized persons)</span><span class="font2">. Kata </span><span class="font2" style="font-style:italic;">cryptographi</span><span class="font2"> berasal dari kata Yunani </span><span class="font2" style="font-style:italic;">kryptos </span><span class="font2">(tersembunyi) dan </span><span class="font2" style="font-style:italic;">graphein</span><span class="font2"> (menulis). </span><span class="font2" style="font-style:italic;">Cryptanalysis</span><span class="font2"> adalah prosedur untuk melakukan mekanisme kriptografi dengan cara mendapatkan </span><span class="font2" style="font-style:italic;">plaintext</span><span class="font2"> atau kunci dari </span><span class="font2" style="font-style:italic;">ciphertext</span><span class="font2"> yang digunakan untuk menghasilkan informasi berharga kemudian mengubah atau memalsukan pesan dengan tujuan untuk menipu penerima yang sesungguhnya [6].</span></p>
<p><span class="font2">Prinsip - prinsip yang mendasari kriptografi antara lain </span><span class="font2" style="font-style:italic;">Confidelity</span><span class="font2"> (kerahasiaan), Data </span><span class="font2" style="font-style:italic;">ntegrity</span><span class="font2"> (keutuhan data), </span><span class="font2" style="font-style:italic;">Authentication</span><span class="font2"> (keotentikan), dan </span><span class="font2" style="font-style:italic;">Non - repudiation</span><span class="font2"> (anti -penyangkalan). konsep yang sering digunakan dalam bidang kriptografi :</span></p>
<ul style="list-style:none;"><li>
<p><span class="font2">1) &nbsp;&nbsp;&nbsp;Plaintext (M) adalah data atau file yang hendak dikirimkan (berisi data atau file asli).</span></p></li>
<li>
<p><span class="font2">2) &nbsp;&nbsp;&nbsp;Ciphertext (C) adalah data atau file ter-enkrip (tersandi) yang merupakan hasil enkripsi.</span></p></li>
<li>
<p><span class="font2">3) &nbsp;&nbsp;&nbsp;Enkripsi (fungsi E) adalah proses pengubahan plaintext menjadi ciphertext.</span></p></li>
<li>
<p><span class="font2">4) &nbsp;&nbsp;&nbsp;Dekripsi (fungsi D) adalah proses pengubahan ciphertext menjadi plaintext, sehingga berupa data atau file asli [7].</span></p></li></ul>
<ul style="list-style:none;"><li>
<h3><a name="bookmark11"></a><span class="font2" style="font-weight:bold;"><a name="bookmark12"></a>3.2. &nbsp;&nbsp;&nbsp;Pengertian RC 4</span></h3></li></ul>
<p><span class="font2" style="font-style:italic;">RC4</span><span class="font2"> merupakan sistem sandi </span><span class="font2" style="font-style:italic;">stream</span><span class="font2"> berorientasi </span><span class="font2" style="font-style:italic;">byte.</span><span class="font2"> Masukan algoritma enkripsi </span><span class="font2" style="font-style:italic;">RC4 </span><span class="font2">merupakan sebuah </span><span class="font2" style="font-style:italic;">byte,</span><span class="font2"> kemudian dilakukan operasi </span><span class="font2" style="font-style:italic;">XOR</span><span class="font2"> dengan sebuah </span><span class="font2" style="font-style:italic;">byte</span><span class="font2"> kunci, dan menghasilkan sebuah </span><span class="font2" style="font-style:italic;">byte</span><span class="font2"> sandi [5]. </span><span class="font2" style="font-style:italic;">RC4</span><span class="font2"> merupakan algoritma kunci simetris berbentuk </span><span class="font2" style="font-style:italic;">stream cipher</span><span class="font2"> yang digunakan untuk memproses unit atau input data, pesan atau informasi pada suatu saat. Unit atau data pada dasarnya merupakan sebuah </span><span class="font2" style="font-style:italic;">byte</span><span class="font2"> atau bahkan bit [6].</span></p>
<p><span class="font2">Keamanan merupakan aspek yang penting untuk sistem informasi. Bermacam cara untuk menjaga sebuah sistem agar kerahasiaan dari informasi tetap terjaga. Pentingnya informasi hanya boleh diketahui oleh pihak-pihak yang berwenang. Berpindahnya informasi ke tangan pihak lain yang tidak diharapkan dapat mengakibatkan kerugian dan kehilangan bagi pemilik informasi [8].</span></p>
<p><span class="font2" style="font-style:italic;">Database</span><span class="font2"> adalah sebuah system yang berfungsi untuk menyimpan dan mengolah sekumpulan data. Setiap </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> mempunyai API tertentu untuk membuat, mengakses, mengatur, mencari, dan menyalin data yang ada di dalamnya sehingga bisa dimanfaatkan oleh aplikasinya. Untuk menampung dan mengatur data yang begitu banyak, dapat menggunakan </span><span class="font2" style="font-style:italic;">Relational Database Management System</span><span class="font2"> (RDMS) [9].</span></p>

<ul style="list-style:none;"><li>
<p><span class="font2">a) Algoritma rivest code 4</span></p></li></ul>
<p><span class="font2" style="font-style:italic;">Rivest code</span><span class="font2"> 4 (RC4) merupakan salah satu algoritma kriptografi modern kunci simetris berbentuk </span><span class="font2" style="font-style:italic;">stream chipper,</span><span class="font2"> sehingga dalam memproses data atau informasi pada satu saat serta menggunakan dua buah </span><span class="font2" style="font-style:italic;">substitution box</span><span class="font2"> (s-</span><span class="font2" style="font-style:italic;">box</span><span class="font2">) berupa </span><span class="font2" style="font-style:italic;">array</span><span class="font2"> dengan panjang permutasi 256 dan S-Box kedua, yang berisi permutasi merupakan fungsi dari kunci dengan panjang yang variabel. Langkah-langkah proses algoritma </span><span class="font2" style="font-style:italic;">Rivest code</span><span class="font2"> 4 yaitu inisialisasi S-Box pertama, S[0],S[1],...,S[255], dengan bilangan 0 sampai 255. Pertama isi secara berurutan S [0] = 0, S[1] = 1,..., S[255] = 255. Selanjutnya di inisialisasikan dengan array S-Box yang lain, sebagai contoh array B dengan panjang 256. Isi array B dengan kunci public yang diulangi sampai seluruh proses pengisian array B[0], B[1],...,B[255] terisi seluruhnya. Proses menginisialisasi S-Box (Array S) </span><span class="font6">for i = 0 to 255</span></p>
<p><span class="font6">S[i] = i</span></p>
<p><span class="font2">Proses menginisialisasi S-Box (Array B)</span></p>
<p><span class="font2">Array Kunci yang mempunyai panjang “</span><span class="font2" style="font-style:italic;">length</span><span class="font2">”. </span><span class="font6">for i = 0 to 255</span></p>
<p><span class="font6">B[i] = Kunci[i mod length]</span></p>
<p><span class="font2">Langkah Selanjutnya yaitu pengacakan S-Box dengan cara sebagai berikut: </span><span class="font6">i = 0 &nbsp;; j = 0</span></p>
<p><span class="font6">for i = 0 to 255 { j = (j+S[i]+B[i]) mod 256</span></p>
<p><span class="font6">swap S[i] dan S[j] }</span></p>
<p><span class="font2">Langkah berikutnya, membuat byte acak dengan cara sebagai berikut: </span><span class="font6">i = (i+1) mod 256</span></p>
<p><span class="font6">j = (j+S[i]) mod 256</span></p>
<p><span class="font6">swap S[i] dan S[j]</span></p>
<p><span class="font6">t = (S[i]+S[j]) mod 256</span></p>
<p><span class="font6">B = S[t]</span></p>
<p><span class="font2">Byte B di-XOR-kan dengan plainteks guna mendapatkan cipherteks atau di-XOR-kan dengan cipherteks guna mendapatkan plainteks.</span></p>

<p><span class="font2">Pada Gambar 12 memperlihatkan data yang di tampilkan dari tabel barang keluar sudah</span></p>
<p><span class="font2">terdekripsi.</span></p>
<p><span class="font2">Pengujian sistem dilaksanakan guna memastikan sistem informasi yang telah dibuat sesuai</span></p>
<p><span class="font2">dengan desain atau rancangannya. Pengujian penggunaan algoritma RC4 pada sistem</span></p>
<p><span class="font2">informasi </span><span class="font2" style="font-style:italic;">inventory stock</span><span class="font2"> barang dilaksanakan menggunakan metode </span><span class="font2" style="font-style:italic;">Black Box Testing</span><span class="font2"> [10]. Metode </span><span class="font2" style="font-style:italic;">Black Box Testing</span><span class="font2"> yang merupakan salah satu pengujian sistem yang memfokuskan</span></p>
<p><span class="font2">proses pengujiannya pada data yang disimpan dan ditampilkan pada sistem informasi tersebut. Penggunaan metode </span><span class="font2" style="font-style:italic;">Black Box Testing</span><span class="font2"> untuk mengidentifikasi dan mengetahui proses penggunaan algoritma rivest code 4 dapat memenuhi perancangan yang telah dibuat yaitu menjaga kerahasiaan dan keamanan data yang disimpan di dalam </span><span class="font2" style="font-style:italic;">database</span><span class="font2">. Pengujian penggunaan algoritma RC4 pada sistem informasi koperasi </span><span class="font2" style="font-style:italic;">inventory stock</span><span class="font2"> barang ini antara lain:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font2">1. &nbsp;&nbsp;&nbsp;Halaman </span><span class="font2" style="font-style:italic;">login,</span><span class="font2"> dilakukan proses pengujian antara lain dengan memasukkan data pada </span><span class="font2" style="font-style:italic;">form login</span><span class="font2">,selanjutnya proses </span><span class="font2" style="font-style:italic;">login</span><span class="font2"> berhasil walaupun data yang di digunakan untuk </span><span class="font2" style="font-style:italic;">login </span><span class="font2">tidak di enkripsi padahal data pada tabel admin di enkripsi.</span></p></li>
<li>
<p><span class="font2">2. &nbsp;&nbsp;&nbsp;Halaman data barang, dilakukan proses pengujian antara lain data barang yang ditampilkan berhasil di dekripsi; data barang yang di simpan pada </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> berhasil di enkripsi; detail data barang yang ditampilkan berhasil di dekripsi; proses ubah data barang berhasil, data yang di ubah berhasil di simpan dalam bentuk enkripsi dan yang di tampilkan dalam bentuk dekripsi; data barang berhasil dihapus; cetak data barang berhasil dan pencarian data barang berhasil.</span></p></li>
<li>
<p><span class="font2">3. &nbsp;&nbsp;&nbsp;Halaman data karyawan, dilakukan proses pengujian antara lain data karyawan yang ditampilkan berhasil di dekripsi; data karyawan yang di simpan pada </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> berhasil di enkripsi; detail data karyawan yang ditampilkan berhasil di dekripsi; proses ubah data karyawan berhasil, data yang di ubah berhasil di simpan dalam bentuk enkripsi dan yang di tampilkan dalam bentuk dekripsi; data karyawan berhasil dihapus; cetak data karyawan berhasil dan pencarian data karyawan berhasil</span></p></li>
<li>
<p><span class="font2">4. &nbsp;&nbsp;&nbsp;Halaman data kategori barang, dilakukan proses pengujian antara lain data kategori barang yang ditampilkan berhasil di dekripsi; data kategori barang yang di simpan pada </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> berhasil di enkripsi; detail data kategori barang yang ditampilkan berhasil di dekripsi; proses ubah data kategori barang berhasil, data yang di ubah berhasil di simpan dalam bentuk enkripsi dan yang di tampilkan dalam bentuk dekripsi; data kategori barang berhasil dihapus; cetak data kategori barang berhasil dan pencarian data kategori barang berhasil.</span></p></li>
<li>
<p><span class="font2">5. &nbsp;&nbsp;&nbsp;Halaman data admin, dilakukan proses pengujian antara lain data admin yang ditampilkan berhasil di dekripsi; data admin yang di simpan pada </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> berhasil di enkripsi; detail data admin yang ditampilkan berhasil di dekripsi; proses ubah data admin berhasil, data yang di ubah berhasil di simpan dalam bentuk enkripsi dan yang di tampilkan dalam bentuk dekripsi; data admin berhasil dihapus</span></p></li>
<li>
<p><span class="font2">6. &nbsp;&nbsp;&nbsp;Halaman data barang masuk, dilakukan proses pengujian antara lain data barang masuk yang ditampilkan berhasil di dekripsi; data barang masuk yang di simpan pada </span><span class="font2" style="font-style:italic;">database </span><span class="font2">berhasil di enkripsi; detail data barang masuk yang ditampilkan berhasil di dekripsi; proses ubah data barang masuk berhasil, data yang di ubah berhasil di simpan dalam bentuk enkripsi dan yang di tampilkan dalam bentuk dekripsi; data barang masuk berhasil dihapus; cetak data barang masuk berhasil dan pencarian data barang masuk berhasil</span></p></li>
<li>
<p><span class="font2">7. &nbsp;&nbsp;&nbsp;Halaman data barang keluar, dilakukan proses pengujian antara lain data barang keluar yang ditampilkan berhasil di dekripsi; data barang keluar yang di simpan pada </span><span class="font2" style="font-style:italic;">database </span><span class="font2">berhasil di enkripsi; detail data barang keluar yang ditampilkan berhasil di dekripsi; proses ubah data barang keluar berhasil, data yang di ubah berhasil di simpan dalam bentuk enkripsi dan yang di tampilkan dalam bentuk dekripsi; data barang keluar berhasil dihapus; cetak data barang keluar berhasil dan pencarian data barang keluar berhasil.</span></p></li></ul>
<p><span class="font2">Sistem informasi </span><span class="font2" style="font-style:italic;">inventory stock</span><span class="font2"> barang yang diusulkan pada makalah ini memiliki beberapa perbedaan dengan sistem informasi </span><span class="font2" style="font-style:italic;">stock</span><span class="font2"> barang yang lain yang sudah pernah dibuat, yaitu data yang disimpan di dalam </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> di enkripsi menggunakan algoritma RC4 dan data yang ditampilakan di dekripsi ke dalam bentuk semula, sedangkan Sistem informasi </span><span class="font2" style="font-style:italic;">inventory</span><span class="font2"> yang datanya disimpan dalam bentuk asli atau tidak di enkripsi di antaranya adalah perancangan sistem informasi inventori barang pada cv. iltizam </span><span class="font2" style="font-style:italic;">cooperation</span><span class="font2"> [11], sistem inventori barang dengan teknologi ajax [12], sistem informasi </span><span class="font2" style="font-style:italic;">inventory</span><span class="font2"> alat tulis kantor (ATK) menggunakan metode waterfall [13].</span></p>

<h3><a name="bookmark35"></a><span class="font2" style="font-weight:bold;"><a name="bookmark36"></a>5. Kesimpulan</span></h3></li></ul>
<p><span class="font2">Berdasarkan keseluruhan pengujian yang telah dilaksanakan, data yang di di simpan di dalam </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> pada sistem informasi </span><span class="font2" style="font-style:italic;">inventory stock</span><span class="font2"> barang pada distributor PT. </span><span class="font2" style="font-style:italic;">Wings Food </span><span class="font2">berhasil di enkripsi sehingga terjamin kerahasiaan dan keamanannya. Sistem informasi </span><span class="font2" style="font-style:italic;">inventory stock</span><span class="font2"> barang pada pada distributor PT. </span><span class="font2" style="font-style:italic;">Wings Food</span><span class="font2"> juga telah berhasil mendekripsi data yang sebenarnya pada halaman tampil data ataupun laporan.</span></p>
<p><span class="font2">Sistem-sistem yang dibangun sebelumnya hanya menyimpan data pada </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> dengan bentuk sesuai dengan apa yang dimasukkan pada sistem informasi tersebut, sehingga orang yang melihat isi </span><span class="font2" style="font-style:italic;">database</span><span class="font2"> tersebut dapat membaca artinya dan tidak terjamin keamanannya.</span></p>
