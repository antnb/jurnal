<h1><a name="bookmark0"></a><span class="font8" style="font-weight:bold;"><a name="bookmark1"></a>Analisis Performansi Dan Efisiensi Cloud Computing Pada Sistem Perbankan</span></h1>
<p><span class="font6">I K. Adi Kurniawan<sup>1</sup>, N. Putra Sastra<sup>2</sup>, M. Sudarma<sup>3</sup></span></p>
<p><span class="font4" style="font-style:italic;">Submission: 27-10-2019, Accepted: 25-11-2019</span></p>
<p><span class="font4" style="font-weight:bold;font-style:italic;">Abstract</span><span class="font4" style="font-weight:bold;">— The use of physical servers for Bank Mandiri branch office’s operations has several shortcomings; such as exorbitant server procurement costs, limitation on upgrading parts, high maintenance cost, requirement for land procurement, and excessive electricity consumption. An alternative to that is cloud computing technology based on virtualization using the ROCCA adoption method. In this study, cloud computing was implemented with Microsoft Hyper-V virtualization to run Bank Mandiri branch’s operational applications. The result of performance and cost efficiency tests of cloud computing is then compared to that of physical servers. The test results demonstrate that the ROCCA method can be utilized for cloud computing adoption in the banking sector. Overall, physical server CPU (36% score) works 4 times better than cloud server (9% score). However, cloud server memory (score 50%) works 1.4 times faster than physical servers (score 35%). The performance of the cloud server hard disk (52% score) is 1.1 times better than the physical server (48% score). In terms of access speed, physical server is faster as it reaches a bandwidth of 94.9 Mbits/sec, while cloud server reaches 94.1 Mbits/sec. As to investment efficiency; the cost of a cloud server for one year period is lower than that of a physical server. However, for a period of five years and above; the cloud server costs are higher than that of physical servers.</span></p>
<p><span class="font4" style="font-weight:bold;font-style:italic;">Abstrak</span><span class="font4" style="font-weight:bold;">— Penggunaan server fisik untuk operasional kantor cabang Bank Mandiri memiliki beberapa kekurangan, yaitu biaya pengadaan server yang tinggi, keterbatasan </span><span class="font4" style="font-weight:bold;font-style:italic;">upgrade part</span><span class="font4" style="font-weight:bold;">, biaya </span><span class="font4" style="font-weight:bold;font-style:italic;">maintainance</span><span class="font4" style="font-weight:bold;">, kebutuhan lahan dan konsumsi listrik. Alternatif yang bisa digunakan adalah teknologi </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud computing </span><span class="font4" style="font-weight:bold;">berbasis virtualisasi dengan metoda adopsi ROCCA. Pada penelitian ini implementasi </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud computing</span><span class="font4" style="font-weight:bold;"> menggunakan virtualisasi Microsoft Hyper-V. Selanjutnya dilakukan komparasi performansi dan efisiensi biaya yang dibandingkan dengan implementasi server fisik. Hasil pengujian menunjukkan metoda ROCCA dapat digunakan untuk adopsi </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud computing </span><span class="font4" style="font-weight:bold;">pada sektor perbankan. Performansi CPU server fisik (skor 36%) bekerja 4 kali lebih baik daripada server </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud</span><span class="font4" style="font-weight:bold;"> (skor 9%). Performansi memori server </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud</span><span class="font4" style="font-weight:bold;"> (skor 50%) bekerja 1,4 kali lebih cepat daripada server fisik (skor 35%). Sedangkan performansi </span><span class="font4" style="font-weight:bold;font-style:italic;">disk</span><span class="font4" style="font-weight:bold;"> server </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud</span><span class="font4" style="font-weight:bold;"> (skor 52%) lebih cepat 1,1 kali daripada server fisik (skor 48%). Pada aspek kecepatan akses, server fisik lebih cepat dengan </span><span class="font4" style="font-weight:bold;font-style:italic;">bandwidth</span><span class="font4" style="font-weight:bold;"> 94,9 Mbits/detik, sedangkan server </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud</span><span class="font4" style="font-weight:bold;"> 94,1 Mbits/detik. Dari sisi efisiensi investasi, selama jangka waktu satu tahun biaya server </span><span class="font4" style="font-weight:bold;font-style:italic;">cloud </span><span class="font4" style="font-weight:bold;">lebih rendah daripada server fisik. Namun untuk jangka waktu lima tahun keatas, biaya server cloud lebih tinggi daripada server fisik.</span></p>
<p><span class="font5">Saat ini operasional perbankan di kantor cabang Bank Mandiri menggunakan aplikasi </span><span class="font5" style="font-style:italic;">desktop</span><span class="font5"> dengan server yang ditempatkan pada masing-masing kantor cabang. Arsitektur berbasis server fisik ini memiliki kekurangan, yaitu:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font7">1.</span><span class="font5"> &nbsp;&nbsp;&nbsp;Biaya pengadaan server tinggi dan butuh waktu lama.</span></p></li>
<li>
<p><span class="font5">2. &nbsp;&nbsp;&nbsp;Kebutuhan setiap kantor cabang berbeda namun spesifikasi server sama untuk seluruh kantor cabang. Di sisi lain terdapat kendala skalibilitas dimana untuk kebutuhan </span><span class="font5" style="font-style:italic;">upgrade</span><span class="font5"> perlu proses pengadaan kembali yang memakan waktu.</span></p></li>
<li>
<p><span class="font5">3. &nbsp;&nbsp;&nbsp;Perlu adanya </span><span class="font5" style="font-style:italic;">maintainance</span><span class="font5"> server secara berkala, seperti </span><span class="font5" style="font-style:italic;">restart</span><span class="font5"> mesin dan </span><span class="font5" style="font-style:italic;">cleansing</span><span class="font5"> data.</span></p></li>
<li>
<p><span class="font5">4. &nbsp;&nbsp;&nbsp;Dibutuhkan lokasi khusus untuk ruang server dengan standar kelengkapan ruang server serta sumber daya listrik.</span></p></li></ul>
<p><span class="font5">Alternatif solusi untuk kendala tersebut adalah dengan implementasi </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5">. </span><span class="font5" style="font-style:italic;">Cloud computing</span><span class="font5"> merupakan tren baru dibidang komputasi terdistribusi dimana berbagai pihak dapat mengembangkan aplikasi dan layanan berbasis </span><span class="font5" style="font-style:italic;">Service Oriented Architecture</span><span class="font5"> di jaringan internet [1]. </span><span class="font5" style="font-style:italic;">Cloud </span><span class="font5">menjadikan internet sebagai pusat server untuk data dan aplikasi [2]. IaaS (</span><span class="font5" style="font-style:italic;">Infrastructure as a Service</span><span class="font5">) merupakan layanan </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5"> untuk memenuhi kebutuhan infrastruktur seperti virtualisasi, </span><span class="font5" style="font-style:italic;">hardware</span><span class="font5"> server, media penyimpanan, dan jaringan [3]. Selain memberikan efisiensi bagi perusahaan, </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5"> mampu mengingkatkan produktifitas kerja [4]. </span><span class="font5" style="font-style:italic;">Cloud computing</span><span class="font5"> juga mampu memberikan efisiensi dalam hal manajemen sumber daya (fleksibilitas dan skalabilitas) [5]. Semakin banyak industri yang beralih ke teknologi </span><span class="font5" style="font-style:italic;">cloud </span><span class="font5">karena secara efisien mampu meningkatkan kualitas layanan dengan menekan biaya berlebih, mengurangi </span><span class="font5" style="font-style:italic;">downtime</span><span class="font5">, dan pengembangan infrastruktur secara otomasi [6]. Secara umum, </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> memberikan manfaat bagi ekonomi, organisasi, dan sosial [7].</span></p>
<p><span class="font5">Penerapan teknologi informasi (TI) pada sektor perbankan utamanya dipengaruhi oleh bertambahnya kebutuhan nasabah serta peningkatan volume transaksi. Industri perbankan sangat intensif dalam menggunakan TI sehingga sangat bergantung pada perkembangan TI tersebut [8]. Bank merupakan segmen penting dalam area bisnis yang menjadi target </span><span class="font5" style="font-style:italic;">cloud computing.</span><span class="font5"> [9] Bank yang menggunakan </span><span class="font5" style="font-style:italic;">cloud computing </span><span class="font5">memiliki posisi yang lebih baik untuk menanggapi ketidakpastian ekonomi dan sistem keuangan global yang saling berhubungan [10]. </span><span class="font5" style="font-style:italic;">Cloud computing</span><span class="font5"> dapat menghemat biaya operasional yang dikeluarkan karena semakin banyaknya pilihan layanan </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> oleh berbagai vendor [12]. Ada beberapa model pengembangan </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> yang dapat diterapkan pada bidang perbankan, yaitu </span><span class="font5" style="font-style:italic;">private cloud</span><span class="font5">, </span><span class="font5" style="font-style:italic;">community cloud</span><span class="font5">, </span><span class="font5" style="font-style:italic;">public cloud</span><span class="font5">, dan </span><span class="font5" style="font-style:italic;">hybrid cloud</span><span class="font5">. Namun demikian model yang paling sesuai adalah </span><span class="font5" style="font-style:italic;">private cloud</span><span class="font5"> [13]. </span><span class="font5" style="font-style:italic;">Private cloud</span><span class="font5"> memberikan manfaat p-ISSN:1693 – 2951; e-ISSN: </span><span class="font0">2503-2372</span></p>
<p><span class="font5">yang lebih baik dari sisi keamanan dan privasi. Pengembangan </span><span class="font5" style="font-style:italic;">private cloud</span><span class="font5"> dilakukan secara eksklusif oleh organisasi dan hanya dapat diakses oleh internal organisasi atau rekanan yang diberikan kewenangan [14].</span></p>
<p><span class="font5">Penting bagi perusahaan untuk memahami bahwa implementasi </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> bukan hanya terkait dengan hal teknis, namun juga melibatkan tantangan lain berupa kendala biaya, kesiapan pengguna, dan organisasi [15]. Penelitian lain menyebutkan terdapat empat fokus area yang harus diperhatikan sebelum melakukan adopsi </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5">, yaitu aspek teknikal, organisasi, biaya, dan lingkungan [16]. Sebuah organisasi yang akan menerapkan </span><span class="font5" style="font-style:italic;">cloud computing </span><span class="font5">perlu menjalani 5 tahap (fase) adopsi, yaitu tahap </span><span class="font5" style="font-style:italic;">analysis</span><span class="font5">, </span><span class="font5" style="font-style:italic;">planning</span><span class="font5">, </span><span class="font5" style="font-style:italic;">adoption</span><span class="font5">, </span><span class="font5" style="font-style:italic;">migration</span><span class="font5">, dan </span><span class="font5" style="font-style:italic;">management</span><span class="font5"> [17] sebagaimana ditampilkan pada gambar 1 berikut.</span></p><img src="https://jurnal.harianregional.com/media/53917-1.jpg" alt="" style="width:258pt;height:41pt;">
<p><span class="font3">Gambar 1. Tahapan Adopsi ROCCA</span></p>
<p><span class="font5">Tahap </span><span class="font5" style="font-style:italic;">analysis</span><span class="font5"> adalah tahapan untuk mengidentifikasi kebutuhan pemakai, kondisi organisasi, kebijakan dan regulasi serta aspek lain yang memengaruhi keputusan organisasi dalam menerapkan teknologi informasi. Pada fase ini dilakukan dua kegiatan yaitu pengumpulan data dan analisa data [18]. Pada tahap </span><span class="font5" style="font-style:italic;">planning</span><span class="font5">, ditentukan tolok ukur permasalahan keamanan, hukum dan kepatuhan. Tolok ukur ini harus dapat merefleksikan </span><span class="font5" style="font-style:italic;">best practice</span><span class="font5">, kebijakan, dan standar dalam organisasi dan bagaimana ini semua dapat dicapai setelah migrasi ke </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5">. Tahap </span><span class="font5" style="font-style:italic;">adoption </span><span class="font5">merupakan persiapan kebutuhan infrastruktur baik </span><span class="font5" style="font-style:italic;">hardware </span><span class="font5">dan </span><span class="font5" style="font-style:italic;">software</span><span class="font5">. Pada tahap </span><span class="font5" style="font-style:italic;">migration</span><span class="font5"> dilakukan pemindahan data ke server </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5">. Tahap </span><span class="font5" style="font-style:italic;">management</span><span class="font5"> meliputi serah terima dengan </span><span class="font5" style="font-style:italic;">user</span><span class="font5">, kesepakatan </span><span class="font5" style="font-style:italic;">service level agreement</span><span class="font5">, jasa </span><span class="font5" style="font-style:italic;">support</span><span class="font5"> serta dokumentasi.</span></p>
<p><span class="font5">Virtualisasi</span></p>
<p><span class="font5">Virtualisasi merupakan teknologi penggerak dibelakang </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> [19]. Dengan virtualisasi, pengguna layanan </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> dapat dengan mudah melakukan </span><span class="font5" style="font-style:italic;">upgrade </span><span class="font5">kapasitas server [20]. Terdapat dua pendekatan virtualisasi [21], yaitu:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font5">1. &nbsp;&nbsp;&nbsp;Full Virtualization</span></p></li></ul>
<p><span class="font5">Seluruh sistem komputer </span><span class="font5" style="font-style:italic;">host</span><span class="font5"> dibuat dalam kontruksi </span><span class="font5" style="font-style:italic;">software</span><span class="font5"> yang dapat dieksekusi oleh sistem operasi </span><span class="font5" style="font-style:italic;">guest</span><span class="font5"> dimana setiap perintah diterjemahkan menjadi instruksi biner.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font5">2. &nbsp;&nbsp;&nbsp;Para Virtualization</span></p></li></ul>
<p><span class="font5">Terdapat modifikasi pada sistem operasi </span><span class="font5" style="font-style:italic;">guest</span><span class="font5"> berupa penyesuaian kernel sehingga permintaan alokasi </span><span class="font5" style="font-style:italic;">hardware</span><span class="font5"> tidak membutuhkan translasi ke dalam instruksi biner.</span></p>
<ul style="list-style:none;"><li>
<h3><a name="bookmark2"></a><span class="font5"><a name="bookmark3"></a>II.</span><span class="font5" style="font-variant:small-caps;"> &nbsp;&nbsp;&nbsp;Metode Penelitian</span></h3></li></ul>
<p><span class="font5">Pada penelitian ini, dibangun dua infrastruktur server kantor cabang Bank Mandiri, yaitu server fisik sebagaimana kondisi semula dan </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> yang berbasis virtualisasi menggunakan </span><span class="font5" style="font-style:italic;">hypervisor</span><span class="font5"> Windows Hyper-V dengan metode adopsi </span><span class="font5" style="font-style:italic;">Roadmap for Cloud Computing Adoption</span><span class="font5"> (ROCCA).</span></p>
<p><span class="font5">Metode ROCCA diawali dengan tahap </span><span class="font5" style="font-style:italic;">analysis</span><span class="font5"> untuk pengumpulan data melalui wawancara dengan pegawai Bank Mandiri, observasi di kantor cabang untuk mendata perangkat TI, aplikasi yang digunakan, serta dokumen tentang regulasi, standar dan kebijakan yang berlaku. Analisis juga dilakukan pada penyedia layanan </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> di Indonesia dan </span><span class="font5" style="font-style:italic;">top leader</span><span class="font5"> global. Tahap </span><span class="font5" style="font-style:italic;">planning</span><span class="font5"> merupakan tahap perancangan yang meliputi pemilihan teknologi virtualisasi serta anggaran yang dibutuhkan. Hal krusial yang juga diputuskan pada tahap </span><span class="font5" style="font-style:italic;">planning</span><span class="font5"> adalah penentuan aplikasi yang akan dimigrasi. Tahap selanjutnya adalah </span><span class="font5" style="font-style:italic;">adoption</span><span class="font5">, yaitu mempersiapkan semua kebutuhan insfrastruktur, meliputi kebutuhan </span><span class="font5" style="font-style:italic;">hardware </span><span class="font5">dan </span><span class="font5" style="font-style:italic;">software</span><span class="font5"> seperti sistem operasi, </span><span class="font5" style="font-style:italic;">virtual machine manager</span><span class="font5">, aplikasi </span><span class="font5" style="font-style:italic;">database</span><span class="font5">, dan aplikasi operasional kantor cabang (</span><span class="font5" style="font-style:italic;">Branch Delivery System)</span><span class="font5">. Tahap </span><span class="font5" style="font-style:italic;">migration</span><span class="font5"> merupakan inti dari adopsi arsitektur </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5">, karena pada tahap ini dilakukan pemindahan data dari server fisik ke server </span><span class="font5" style="font-style:italic;">cloud.</span><span class="font5"> Pada tahap </span><span class="font5" style="font-style:italic;">management</span><span class="font5"> dilakukan pengujian untuk memastikan sistem berjalan normal sebagaimana pada saat sebelum migrasi. <p><span class="font5">A. &nbsp;&nbsp;&nbsp;Implementasi ROCCA</span></p></li></ul>
<ul style="list-style:none;"><li>
<p><span class="font5">1. &nbsp;&nbsp;&nbsp;Tahap Analysis</span></p></li></ul>
<p><span class="font5">Secara topologi, saat ini server kantor cabang Bank Mandiri terhubung dengan </span><span class="font5" style="font-style:italic;">client</span><span class="font5"> melalui koneksi </span><span class="font5" style="font-style:italic;">Local Area Network</span><span class="font5"> (LAN). <h3><a name="bookmark24"></a><span class="font5"><a name="bookmark25"></a>IV.</span><span class="font5" style="font-variant:small-caps;"> &nbsp;&nbsp;&nbsp;Kesimpulan</span></h3></li></ul>
<p><span class="font5">Berdasarkan implementasi sistem </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> diatas, dapat ditarik beberapa kesimpulan sebagai berikut:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font5">1. &nbsp;&nbsp;&nbsp;Arsitektur server fisik yang diterapkan pada sistem perbankan dapat dimigrasi ke arsitektur </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5">.</span></p></li>
<li>
<p><span class="font5">2. &nbsp;&nbsp;&nbsp;Metoda ROCCA sebagai salah satu metoda adopsi </span><span class="font5" style="font-style:italic;">cloud computing</span><span class="font5"> dapat diterapkan dalam migrasi sistem perbankan.</span></p></li>
<li>
<p><span class="font5">3. &nbsp;&nbsp;&nbsp;Performansi CPU pada server </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5"> memberikan keunggulan pada proses </span><span class="font5" style="font-style:italic;">single thread</span><span class="font5">, namun pada proses </span><span class="font5" style="font-style:italic;">multi thread</span><span class="font5"> server fisik meberikan performansi lebih baik.</span></p></li>
<li>
<p><span class="font5">4. &nbsp;&nbsp;&nbsp;Performansi memory server </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5"> memberikan hasil yang lebih baik karena server </span><span class="font5" style="font-style:italic;">cloud</span><span class="font5"> berbasis virtualisasi dengan </span><span class="font5" style="font-style:italic;">hypervisor</span><span class="font5"> (</span><span class="font5" style="font-style:italic;">virtual machine &nbsp;manager</span><span class="font5">),</span></p></li></ul>
