<h1><a name="bookmark0"></a><span class="font4" style="font-weight:bold;"><a name="bookmark1"></a>Rancang Bangun Aplikasi “Collaborative Project Scheduler” Berbasis </span><span class="font4" style="font-weight:bold;font-style:italic;">Mobile Web</span></h1>
<p><span class="font3" style="font-weight:bold;">I Gusti Agung Oka Widiarsana, I Made Sukarsa, Putu Wira Buana </span><span class="font3">Jurusan Teknologi Informasi, Fakultas Teknik, Universitas Udayana </span><a href="mailto:oka.widiarsana@gmail.com"><span class="font3">oka.widiarsana@gmail.com</span></a><span class="font3">, </span><a href="mailto:sukarsa@ee.unud.ac.id"><span class="font3">sukarsa@ee.unud.ac.id</span></a><span class="font3">, </span><a href="mailto:wirabuana@it.unud.ac.id"><span class="font3">wirabuana@it.unud.ac.id</span></a></p>
<p><span class="font3" style="font-weight:bold;font-style:italic;">Abstrak</span></p>
<p><span class="font3" style="font-style:italic;">Kegagalan pencapaian target pada proyek dapat terjadi karena kurangnya perencanaan serta lemahnya pengawasan di dalam proses pengerjaan proyek. Perkembangan teknologi saat ini khususnya web memungkinkan untuk mengembangkan aplikasi manajemen proyek berbasisweb.Collaborative Project Scheduler merupakan sistem manajemen proyek berbasis websederhana, sehingga mudah untuk dipahami dan secara umum sesuai standar konsep manajemen proyek dapat membantu pengguna dalam menangani proyek. Aplikasi dikembangkan menggunakan HTML5, dengan konsep responsive design sehingga aplikasi dapat diakses dengan berbagai perangkat. Pertukaran data pada aplikasi ditunjang dengan AJAX Push sehingga pertukaran data komunikasi seperti chat akan bersifat realtime. Aplikasi dilengkapi dengan beberapa fitur komunikasi sehingga dapat menunjang keperluan komunikasi individual maupun kelompok di dalam proses pengerjaan proyek.Pengujian menggunakan kuesioner yang berisi serangkaian pernyataan mengenai kepuasan penggunaan dan penilaian aplikasi yang diisi oleh 36 responden memberikan persentasi hasil 95,37% yang termasuk pada kategori sangat baik.</span></p>
<p><span class="font3" style="font-weight:bold;">Kata Kunci</span><span class="font3">: Manajemen Proyek, Aplikasi Kolaborasi, </span><span class="font3" style="font-style:italic;">ResponsiveWebsite</span><span class="font3">, </span><span class="font3" style="font-style:italic;">AJAXPush</span><span class="font3">, </span><span class="font3" style="font-style:italic;">HTML5</span></p>
<p><span class="font3" style="font-weight:bold;font-style:italic;">Abstract</span></p>
<p><span class="font3" style="font-style:italic;">Failure to achieve targets in the project may happen due to lack of planning and monitoring in the process of the project. Current technological developments, especially the web allows to develop a web-based project management application.Collaborative Project Scheduler is a simpleweb-based project management system, making it easy to understand and in general according to the standard project management concepts can help the user in handling the project.Applications developed using HTML5, with a responsive design concept so the application can be accessed by various devices. Data exchange on application is supported with the AJAX Push thus the exchange of data communication such as conversation will be real time. The application comes with several features that can support the communication needs of individual and group communication within the project.Tests using a questionnaire that contains a series of statements regarding the use and satisfaction of application assessment completed by 36 respondents gave excellent scoreresults presentation, which reach 95,37% score.</span></p>
<p><span class="font3" style="font-weight:bold;font-style:italic;">Keywords</span><span class="font3" style="font-style:italic;">: Project Management, Collaborative Application, Responsive Website, AJAX Push, HTML5</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark2"></a><span class="font3" style="font-weight:bold;"><a name="bookmark3"></a>1. &nbsp;&nbsp;&nbsp;Pendahuluan</span></h2></li></ul>
<p><span class="font3">Sistem manajemen proyek merupakan teknologi yang secara cepat berkembang di industri IT saat ini. Manajemen proyek sendiri merupakan serangkaian proses dan kegiatan dalam melakukan perencanaan, pengaturan, dan pengendalian atas sumber daya untuk menyelesaikan proyek serta mencapai tujuan yang telah ditetapkan. Dewasa ini kebutuhan akan mobilitas sangat tinggi, demikian juga dengan kebutuhan komunikasi yang kerap menjadi kendala dalam proses pengerjaan proyek, khususnya jika kelompok kerja tidak berada pada satu lokasi. Berdasarkan permasalahan tersebut dibutuhkan sebuah aplikasi manajemen proyek yang memiliki fitur komunikasi yang baik serta dapat diaksesdimana saja dan dapat diakses menggunakan berbagai perangkat termasukperangkat </span><span class="font3" style="font-style:italic;">mobile</span><span class="font3">.Diperlukan juga sebuah sistem yang dapat berperan sebagai wadah untuk organisasi proyek serta data proyek itu sendiri sehingga proyek dapat lebih mudah dipantau proses pengerjaannya.Aplikasi dikembangkan menggunakan HTML5, dengan konsep </span><span class="font3" style="font-style:italic;">responsivedesign</span><span class="font3"> sehingga aplikasi</span></p>
<p><span class="font3">dapat diakses dengan baik melalui berbagai perangkat</span><span class="font3" style="font-style:italic;">mobile</span><span class="font3">.Tampilan aplikasi menjadi tetap proporsional walaupun diakses dengan menggunakan berbagai resolusi perangkat </span><span class="font3" style="font-style:italic;">mobile</span><span class="font3">. Pertukaran data pada aplikasi ditunjang dengan </span><span class="font3" style="font-style:italic;">AJAXPush</span><span class="font3"> sehingga pertukaran data komunikasi seperti </span><span class="font3" style="font-style:italic;">chat</span><span class="font3"> dan </span><span class="font3" style="font-style:italic;">update</span><span class="font3"> proses pekerjaan bersifat </span><span class="font3" style="font-style:italic;">real time.</span><span class="font3"> Pertukaran data juga menjadi lebih efisien karena </span><span class="font3" style="font-style:italic;">server</span><span class="font3"> hanya akan mengirimkan atau meneruskan data ketika ada data baru yang masuk pada </span><span class="font3" style="font-style:italic;">server</span><span class="font3">, hal ini dapat menghemat penggunaan </span><span class="font3" style="font-style:italic;">bandwidth</span><span class="font3"> yang merupakan prioritas bagi pengguna perangkat </span><span class="font3" style="font-style:italic;">mobile</span><span class="font3">.</span></p>

<ul style="list-style:none;"><li>
<h2><a name="bookmark8"></a><span class="font3" style="font-weight:bold;"><a name="bookmark9"></a>2.2. &nbsp;&nbsp;&nbsp;Perancangan Sistem</span></h2></li></ul>
<p><span class="font3">Berikut adalah gambaran umum dari aplikasi </span><span class="font3" style="font-style:italic;">Collaborative Project Scheduler</span><span class="font3">, yang mana secara umum menggambarkan bagaimana aplikasi dapat digunakan.</span></p><img src="https://jurnal.harianregional.com/media/17800-1.jpg" alt="" style="width:301pt;height:159pt;">
<p><span class="font3">Gambar 1. Gambaran umum sistem</span></p>
<p><span class="font3">Aplikasi </span><span class="font3" style="font-style:italic;">Collaborative Project Scheduler</span><span class="font3"> Berbasis </span><span class="font3" style="font-style:italic;">MobileWeb</span><span class="font3">, seperti yang ditunjukkan pada Gambar 1, memungkinkan manajer proyek untuk meng-</span><span class="font3" style="font-style:italic;">assign</span><span class="font3"> atau memberikan pekerjaan-pekerjaan yang terdapat di dalam sebuah proyek kepada </span><span class="font3" style="font-style:italic;">user</span><span class="font3"> lain yang nantinya akan bertanggung jawab untuk menyelesaikan pekerjaan tersebut berdasarkan jangka waktu yang telah ditetapkan sebelumnya.Pemecahan struktur proyek yang dapat dilakukan pada aplikasi untuk sebuah proyek dari tingkat tertinggi sampai tingkat terendah adalah sebagai berikut:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">1. &nbsp;&nbsp;&nbsp;</span><span class="font3" style="font-style:italic;">Project</span><span class="font3">, yaitu proyek itu sendiri yang mana menggambarkan keseluruhan kegiatan</span></p></li></ul>
<p><span class="font3">pekerjaan yang terdapat di dalam sebuah proyek.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">2. &nbsp;&nbsp;&nbsp;</span><span class="font3" style="font-style:italic;">Task</span><span class="font3">, yaitu level berikutnya setelah </span><span class="font3" style="font-style:italic;">project</span><span class="font3">, dimana </span><span class="font3" style="font-style:italic;">task</span><span class="font3"> adalah kumpulan pekerjaan-</span></p></li></ul>
<p><span class="font3">pekerjaan secara garis besar pada proyek. </span><span class="font3" style="font-style:italic;">Task</span><span class="font3"> dapat dikelola oleh </span><span class="font3" style="font-style:italic;">supervisor</span><span class="font3"> atau koordinator </span><span class="font3" style="font-style:italic;">task</span><span class="font3">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">3. &nbsp;&nbsp;&nbsp;</span><span class="font3" style="font-style:italic;">Subtask</span><span class="font3">, yaitu struktur terendah atau detail pekerjaan pada sebuah proyek. </span><span class="font3" style="font-style:italic;">Subtask</span></p></li></ul>
<p><span class="font3">dapat ditugaskan kepada </span><span class="font3" style="font-style:italic;">worker</span><span class="font3">, yang berarti </span><span class="font3" style="font-style:italic;">worker</span><span class="font3"> tersebut bertanggung jawab untuk menyelesaikan </span><span class="font3" style="font-style:italic;">subtask</span><span class="font3"> tersebut.</span></p>
<p><span class="font3">Semua pengguna aplikasi pada dasarnya memiliki hak akses yang sama pada aplikasi. Pengguna dapat menggunakan aplikasi untuk memanajemen proyek yang dimiliki. Perbedaan hak akses untuk </span><span class="font3" style="font-style:italic;">user</span><span class="font3"> terlihat pada proyek itu sendiri dimana pengguna memiliki peran tersendiri pada proyek, adapun peran yang yang terdapat pada aplikasi adalah:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">1. &nbsp;&nbsp;&nbsp;Manajer Proyek, adalah status pengguna yang terdaftar membuat proyek pada aplikasi</span></p></li></ul>
<p><span class="font3">dan memiliki kendali penuh terhadap proyek tersebut.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">2. &nbsp;&nbsp;&nbsp;</span><span class="font3" style="font-style:italic;">Supervisortask</span><span class="font3">, adalah status untuk pengguna yang dipilih oleh manajer proyek untuk</span></p></li></ul>
<p><span class="font3">mengelola </span><span class="font3" style="font-style:italic;">task</span><span class="font3"> pada proyek. </span><span class="font3" style="font-style:italic;">Supervisor</span><span class="font3"> dapat membuat dan menghapus </span><span class="font3" style="font-style:italic;">subtask</span><span class="font3"> pada </span><span class="font3" style="font-style:italic;">task</span><span class="font3"> serta dapat mendistribusikan kolaborator </span><span class="font3" style="font-style:italic;">task</span><span class="font3"> kepada pekerjaan-pekerjaan (</span><span class="font3" style="font-style:italic;">subtask</span><span class="font3">).</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">3. &nbsp;&nbsp;&nbsp;</span><span class="font3" style="font-style:italic;">Worker</span><span class="font3">, adalah status bagi pengguna yang diberikan tanggung jawab untuk mengerjakan</span></p></li></ul>
<p><span class="font3">pekerjaan-pekerjaan (</span><span class="font3" style="font-style:italic;">subtask</span><span class="font3">) pada proyek. </span><span class="font3" style="font-style:italic;">Worker</span><span class="font3"> dapat memberikan komentar dan </span><span class="font3" style="font-style:italic;">updateprogress</span><span class="font3"> pada </span><span class="font3" style="font-style:italic;">subtask</span><span class="font3"> yang dikerjakan. Satu </span><span class="font3" style="font-style:italic;">subtask</span><span class="font3"> dapat dikerjakan oleh beberapa </span><span class="font3" style="font-style:italic;">worker</span><span class="font3"> yang disebut Kolaborator </span><span class="font3" style="font-style:italic;">subtask</span><span class="font3">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">4. &nbsp;&nbsp;&nbsp;Tim Proyek, merupakan kelompok kerja proyek yang terdiri dari seluruh sumber daya</span></p></li></ul>
<p><span class="font3">manusia yang terlibat dalam pengerjaan proyek termasuk manajer proyek, </span><span class="font3" style="font-style:italic;">supervisor </span><span class="font3">dan </span><span class="font3" style="font-style:italic;">worker</span><span class="font3">.</span></p>
<p><span class="font3">Berikut merupakan DFD level 0 dari aplikasi</span><span class="font3" style="font-style:italic;">Collaborative Project Scheduler</span><span class="font3">, dimana terdapat empat proses utama yaitu proses </span><span class="font3" style="font-style:italic;">register</span><span class="font3">, proses </span><span class="font3" style="font-style:italic;">login</span><span class="font3">, proses </span><span class="font3" style="font-style:italic;">message</span><span class="font3"> serta proses </span><span class="font3" style="font-style:italic;">project</span><span class="font3">.</span></p><img src="https://jurnal.harianregional.com/media/17800-2.png" alt="" style="width:437pt;height:267pt;">
<p><span class="font3">Gambar 2.DFD level 0 aplikasi.</span></p>
<p><span class="font3">Gambar 2 menunjukan DFD level 0 dari aplikasi dimana terdapat empat proses utama. Proses pertama adalah proses </span><span class="font3" style="font-style:italic;">register</span><span class="font3">untuk pendaftaran calon member pada aplikasi. <h2><a name="bookmark16"></a><span class="font3" style="font-weight:bold;"><a name="bookmark17"></a>3.1. &nbsp;&nbsp;&nbsp;Manajemen Proyek</span></h2></li></ul>
<p><span class="font3">Manajemen Proyek merupakan suatu proses dan kegiatan untuk merencanakan, mengorganisasi dan mengendalikan sumber daya untuk mencapai tujuan yang spesifik. Proyek merupakan suatu usaha sementara untuk menghasilkan produk yang unik, layanan atau perolehan, dengan waktu pengerjaan yang sudah ditetapkan dan sering kali dibatasi oleh modal dan waktu. Tantangan utama dari pengelolaan proyek adalah untuk mencapai setiap target dari proyek yang dikelola. Batasan utama dari manajemen proyek adalah waktu, kualitas dan biaya</span></p>
<p><span class="font3">yang dikenal sebagai </span><span class="font3" style="font-style:italic;">Project Management Triangle</span><span class="font3"> atau dikenal juga dengan </span><span class="font3" style="font-style:italic;">Iron Triangle </span><span class="font3">dimana nantinya batasan ini akan menentukan kualitas dari proyek itu sendiri[1].</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3" style="font-weight:bold;">3.2.</span><span class="font3" style="font-weight:bold;font-style:italic;"> &nbsp;&nbsp;&nbsp;Responsive Web Design</span></p></li></ul>
<p><span class="font3">Responsif </span><span class="font3" style="font-style:italic;">webdesign</span><span class="font3"> merupakan kegiatan atau usaha untuk membangun sebuah </span><span class="font3" style="font-style:italic;">web</span><span class="font3">site yang nantinya sesuai atau mampu memberi respon untuk berbagai perangkat dengan berbagai ukuran resolusi layar, tidak terbatas seberapa besar atau kecil, perangkat </span><span class="font3" style="font-style:italic;">mobile </span><span class="font3">maupun </span><span class="font3" style="font-style:italic;">desktop</span><span class="font3">.Penerapannya terdiri dari perpaduan </span><span class="font3" style="font-style:italic;">layout</span><span class="font3"> dan </span><span class="font3" style="font-style:italic;">grid</span><span class="font3"> yang fleksibel, gambar dan penerapan dari CSS </span><span class="font3" style="font-style:italic;">mediaqueries.</span><span class="font3">Responsif </span><span class="font3" style="font-style:italic;">web</span><span class="font3"> merupakan disain yang mampu untuk beradaptasi pada</span><span class="font3" style="font-style:italic;">browser</span><span class="font3"> dan </span><span class="font3" style="font-style:italic;">viewport</span><span class="font3"> dari perangkat yang berbeda, berganti </span><span class="font3" style="font-style:italic;">layout</span><span class="font3"> dan konten setiap waktu[2].</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark18"></a><span class="font3" style="font-weight:bold;"><a name="bookmark19"></a>3.3. &nbsp;&nbsp;&nbsp;HTML5 dan CSS</span></h2></li></ul>
<p><span class="font3">HTML5 dibuat dengan tujuan untuk membuat proses </span><span class="font3" style="font-style:italic;">coding</span><span class="font3">menjadi lebih mudah dan logis, serta dapat mendukung teknologi multimedia terbaru, mudah dipahami oleh manusia dan juga lebih mudah dimengerti oleh mesin[3]. HTML5 menempatkan beberapa penekanan pada penyederhaan </span><span class="font3" style="font-style:italic;">markup</span><span class="font3"> yang diperlukan untuk membuat sebuah halaman yang sesuai menurut standar W3C dan menghubungkan semua CSS, </span><span class="font3" style="font-style:italic;">JavaScript</span><span class="font3">, dan </span><span class="font3" style="font-style:italic;">file</span><span class="font3"> gambar. Merampingkan </span><span class="font3" style="font-style:italic;">markup</span><span class="font3"> dapat menjadi suatu yang sangat membantu karena tidak hanya sebatas </span><span class="font3" style="font-style:italic;">responsivedesign</span><span class="font3"> yang merespon </span><span class="font3" style="font-style:italic;">viewport</span><span class="font3">, namun juga menghilangkan </span><span class="font3" style="font-style:italic;">markup</span><span class="font3"> yang berlebih juga membuat waktu akses menjadi lebih cepat dan menghemat </span><span class="font3" style="font-style:italic;">bandwidth.</span></p>
<p><span class="font3" style="font-style:italic;">Cascading Style Sheet</span><span class="font3"> (CSS) merupakan aturan untuk mengendalikan beberapa komponen dalam sebuah </span><span class="font3" style="font-style:italic;">web</span><span class="font3"> sehingga menjadi lebih terstruktur dan seragam. Sama halnya dengan </span><span class="font3" style="font-style:italic;">style</span><span class="font3"> dalam aplikasi pengolah kata seperti </span><span class="font3" style="font-style:italic;">Microsoft Word</span><span class="font3"> yang dapat mengatur beberapa </span><span class="font3" style="font-style:italic;">style</span><span class="font3">, misalnya </span><span class="font3" style="font-style:italic;">heading</span><span class="font3">, </span><span class="font3" style="font-style:italic;">subbab</span><span class="font3">, </span><span class="font3" style="font-style:italic;">body text</span><span class="font3">, </span><span class="font3" style="font-style:italic;">footer</span><span class="font3">, </span><span class="font3" style="font-style:italic;">images</span><span class="font3">, dan lain sebagainya untuk digunakan bersama-sama dalam beberapa berkas.CSS dapat mengendalikan ukuran gambar, warna bagian tubuh pada teks, warna tabel, ukuran </span><span class="font3" style="font-style:italic;">border</span><span class="font3">, warna </span><span class="font3" style="font-style:italic;">border</span><span class="font3">, warna </span><span class="font3" style="font-style:italic;">hyperlink</span><span class="font3">, </span><span class="font3" style="font-style:italic;">mouse over</span><span class="font3">, dan lain sebagainya. CSS merupakan </span><span class="font3" style="font-style:italic;">style sheet</span><span class="font3"> yang digunakan untuk mengatur tampilan dokumen, dengan CSS halaman yang sama dapat ditampilkan dengan format yang berbeda.</span></p>
<ul style="list-style:none;"><li>
<h2><a name="bookmark20"></a><span class="font3" style="font-weight:bold;"><a name="bookmark21"></a>3.4. &nbsp;&nbsp;&nbsp;AJAX</span></h2></li></ul>
<p><span class="font3" style="font-style:italic;">Asynchronous Java Script and XMLHTTP</span><span class="font3">, atau disingkat </span><span class="font3" style="font-style:italic;">AJAX</span><span class="font3">, merupakan suatu teknik pemrograman berbasis </span><span class="font3" style="font-style:italic;">web</span><span class="font3"> untuk menciptakan aplikasi </span><span class="font3" style="font-style:italic;">web</span><span class="font3"> interaktif. Tujuannya adalah untuk memindahkan sebagian besar interaksi pada komputer </span><span class="font3" style="font-style:italic;">web surfer</span><span class="font3">, melakukan pertukaran data dengan </span><span class="font3" style="font-style:italic;">server</span><span class="font3"> di belakang layar, sehingga halaman </span><span class="font3" style="font-style:italic;">web</span><span class="font3"> tidak harus dibaca ulang secara keseluruhan setiap kali seorang pengguna melakukan perubahan dimana ini akan meningkatkan interaktivitas, kecepatan, dan </span><span class="font3" style="font-style:italic;">usability</span><span class="font3">. </span><span class="font3" style="font-style:italic;">Ajax</span><span class="font3"> bukanlah sebuah teknologi, </span><span class="font3" style="font-style:italic;">Ajax </span><span class="font3">merupakan gabungan dari beberapa teknologi, yang berkembang secara mandiri yang yang kemudian hadir bersama dalam cara yang luar biasa[4].</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3" style="font-weight:bold;font-style:italic;">3.5. &nbsp;&nbsp;&nbsp;Ruby on Rails</span></p></li></ul>
<p><span class="font3" style="font-style:italic;">Ruby on Rails</span><span class="font3"> atau yang umumnya hanya disebut </span><span class="font3" style="font-style:italic;">Ruby</span><span class="font3"> merupakan </span><span class="font3" style="font-style:italic;">frameworkweb</span><span class="font3"> yang ditulis di dalam bahasa pemrograman </span><span class="font3" style="font-style:italic;">Ruby</span><span class="font3">.</span><span class="font3" style="font-style:italic;">Ruby on Rails</span><span class="font3"> berkembang dengan pesat menjadi salah satu bahasa yang banyak digunakan dan menjadi sangat terkenal untuk membangun aplikasi </span><span class="font3" style="font-style:italic;">web</span><span class="font3"> yang dinamis sejak dikeluarkan pada tahun 2004[5].</span><span class="font3" style="font-style:italic;">Rubyon Rails</span><span class="font3"> memungkinkan untuk aplikasi dikembangkan dengan cepat dengan menggunakan konsep yang dikenal sebagai </span><span class="font3" style="font-style:italic;">convention over configuration</span><span class="font3">. Beberapa direktori dan </span><span class="font3" style="font-style:italic;">file</span><span class="font3"> dasar aplikasi akan dibuat ketika mulai menggunakan </span><span class="font3" style="font-style:italic;">Ruby on Rails</span><span class="font3"> dan menjalankan </span><span class="font3" style="font-style:italic;">application generator</span><span class="font3">.</span><span class="font3" style="font-style:italic;">Ruby on Rails </span><span class="font3">menggunakan arsitektur MVC (</span><span class="font3" style="font-style:italic;">Model-View-Controller</span><span class="font3">) sebagai dasarnya, arsitektur ini didisain agar bagian-bagian dari aplikasi yang dibuat tetap terpisah saat mengolah data. Keuntungan yang didapat dari arsitektur ini adalah kode program tidak akan tercampur dengan kode tampilan program (</span><span class="font3" style="font-style:italic;">view</span><span class="font3">), kode untuk </span><span class="font3" style="font-style:italic;">controller</span><span class="font3">, serta kode untuk pengolahan data secara langsung (</span><span class="font3" style="font-style:italic;">model</span><span class="font3">), sehingga memudahkan </span><span class="font3" style="font-style:italic;">developer</span><span class="font3"> untuk fokus pada bagian yang dikerjakan.</span></p>
<ul style="list-style:none;"><li>

<h2><a name="bookmark30"></a><span class="font3" style="font-weight:bold;"><a name="bookmark31"></a>5. &nbsp;&nbsp;&nbsp;Kesimpulan</span></h2></li></ul>
<p><span class="font3">Aplikasi </span><span class="font3" style="font-style:italic;">CollaborativeProjectScheduler</span><span class="font3"> dapat menyajikan jadwal dari pengerjaan proyek serta dapat menjabarkan detail pekerjaan-pekerjaan di dalam proyek yang dipecah menjadi level yang lebih rendah pada </span><span class="font3" style="font-style:italic;">task</span><span class="font3"> dan </span><span class="font3" style="font-style:italic;">subtask</span><span class="font3">. Menggunakan teknologi </span><span class="font3" style="font-style:italic;">AJAX Push</span><span class="font3"> dapat meningkatkan efisiensi pengiriman data, dikarenakan </span><span class="font3" style="font-style:italic;">client</span><span class="font3"> (</span><span class="font3" style="font-style:italic;">web browser</span><span class="font3">) hanya perlu mengambil data saat ada data baru yang akan dikirimkan oleh </span><span class="font3" style="font-style:italic;">server</span><span class="font3">, dengan demikian pertukaran data seperti komunikasi dapat berlangsung dengan baik dan bersifat </span><span class="font3" style="font-style:italic;">real time</span><span class="font3">.</span></p>
<p><span class="font3">Memberikan beberapa jalur komunikasi juga memberikan kemudahan tersendiri kepada pengguna aplikasi sehingga dapat berkolaborasi di dalam menyelesaikan proyek.</span></p>
<p><span class="font3">Memanfaatkan teknologi seperti HTML5 dan CSS yang semakin berkembang dapat menciptakan tampilan </span><span class="font3" style="font-style:italic;">website</span><span class="font3"> yang bersifat </span><span class="font3" style="font-style:italic;">responsive</span><span class="font3">.Menggunakan konsep </span><span class="font3" style="font-style:italic;">mobilefirst</span><span class="font3"> dirasa sangat tepat untuk mengembangkan </span><span class="font3" style="font-style:italic;">website</span><span class="font3"> terutama yang ditujukan untuk </span><span class="font3" style="font-style:italic;">mobile</span><span class="font3">, agar tampilan </span><span class="font3" style="font-style:italic;">website</span><span class="font3"> dapat tetap nyaman dilihat saat diakses menggunakan berbagai perangkat. Pengembang tidak lagi perlu membuat banyak</span><span class="font3" style="font-style:italic;">layout</span><span class="font3"> tampilanuntuk beberapa resolusi layar yang berbeda untuk sebuah</span><span class="font3" style="font-style:italic;">website</span><span class="font3">, namun diperlukan perencanaan dan rancangan</span><span class="font3" style="font-style:italic;">interface </span><span class="font3">aplikasi serta </span><span class="font3" style="font-style:italic;">layout</span><span class="font3"> yang sesuai untuk setiap resolusi </span><span class="font3" style="font-style:italic;">device</span><span class="font3">, agar proses pengembangan dapat terlaksana dengan lebih baik di dalam penerapan setiap elemen disain aplikasi.</span></p>
