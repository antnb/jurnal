<p><span class="font7" style="font-weight:bold;font-style:italic;">ABSTRACT</span></p>
<p><span class="font7" style="font-style:italic;">Armijo rule is an inexact line search method to determine step size a</span><span class="font5" style="font-style:italic;"><sub>k</sub> </span><span class="font7" style="font-style:italic;">in some descent method to solve unconstrained local optimization. Modified Armijo was introduced to increase the numerical performance of several descent algorithms that applying this method. The basic difference of Armijo and its modified are in existence of parameter μ</span><span class="font7"> </span><span class="font3">∈</span><span class="font7"> [0,2) </span><span class="font7" style="font-style:italic;">and estimating parameter L</span><span class="font5" style="font-style:italic;"><sub>k</sub> </span><span class="font7" style="font-style:italic;">that be updated in every iteration. This article is comparing numerical solution and time of computation of gradient descent and conjugate gradient hybrid Gilbert-Nocedal (CGHGN) that applying modified Armijo rule. From program implementation in Matlab 6, its known that gradient descent was applying modified Armijo more effectively than CGHGN from one side: iteration needed to reach some norm of gradient</span><span class="font7"> Hg<sub>k</sub> </span><span class="font3">∣∣</span><span class="font7"> </span><span class="font7" style="font-style:italic;">(input by the user). The amount of iteration was representing how long the step size of each algorithm in each iteration. In another side, time of computation has the same conclusion.</span></p>
<p><span class="font7" style="font-weight:bold;font-style:italic;">Keywords</span><span class="font7" style="font-style:italic;">: Armijo line search; conjugate gradient hybrid Gilbert-Nocedal; gradient descent; modified Armijo line search</span></p>
<ul style="list-style:none;"><li>
<h3><a name="bookmark2"></a><span class="font7" style="font-weight:bold;"><a name="bookmark3"></a>1. &nbsp;&nbsp;&nbsp;INTRODUCTION</span></h3></li></ul>
<p><span class="font7">Optimization is still an interesting issue to be learned and developed, especially numerical methods for finding solutions. This is known as numerical optimization. Optimization is the process of finding the optimal solution of a problem either with constraints or without constraints. Numerical optimization allows the discovery of solutions to problems that are difficult to be solved analytically.</span></p>
<p><span class="font7">The optimization problem is essentially a matter of minimizing a function f(x). As for maximization, a function </span><span class="font7" style="font-style:italic;">f(x)</span><span class="font7"> can also be formed as a minimization problem of function -f(x). To solve the minimization problem, some descent iterative methods have been introduced, including gradient descent (steepest descent) and conjugate gradient methods. These two methods are just some of the many methods that can be used to solve local optimization problems. There is much useful application of gradient descent method. One of them is to find the saddle point of a complex integral (Wang, 2008). While Chen (2009) use gradient descent to fit the measured data of laser</span></p>
<p><span class="font7">grain-size distribution of the sediments.</span></p>
<p><span class="font7">While in the conjugate gradient method itself there are various types of parameter determination </span><span class="font7" style="font-style:italic;">β</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7">. In this paper, we will use conjugate gradient hybrid Gilbert-Nocedal which is a hybrid of Fletcher-Reeves conjugate gradient (FR) and Polak-Ribiere-Polyak conjugate gradient (PRP).</span></p>
<p><span class="font7">Gradient descent and conjugate gradient algorithms start with determining the initial point to run the first iteration. The next points will be obtained from each iteration in which the conditions determined will converge to the optimum solution of the optimization objective function. One important part of these two algorithms is the determination of step size </span><span class="font7" style="font-style:italic;">(a</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7" style="font-style:italic;">) </span><span class="font7">on each iteration. This step size determination can be done in many ways called line search. One of the line search rules often used is Armijo rule.</span></p>
<p><span class="font7">To improve the numerical performance of Armijo rules, Shi (2005) have introduced and developed Armijo modification rules. The difference with classical Armijo is the value of parameter </span><span class="font7" style="font-style:italic;">μ </span><span class="font3" style="font-style:italic;">∈</span><span class="font7"> [0,2) and </span><span class="font7" style="font-style:italic;">L</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7"> which is updated on each iteration with a certain estimate. In modified</span></p>
<p><span class="font7">Armijo rules, it is easier to obtain larger step sizes, meaning that with the same stopping criteria, the iterations required by a descent algorithm will be less than using classical Armijo. This will be compared to the gradient descent and conjugate gradient hybrid Gilbert-Nocedal algorithms. Likewise, the time required for each algorithm to solve the optimization problem by applying the classical Armijo rules and its modification in step size determination.</span></p>

<h3><a name="bookmark4"></a><span class="font7" style="font-weight:bold;"><a name="bookmark5"></a>2.1. &nbsp;&nbsp;&nbsp;Gradient Descent</span></h3></li></ul></li></ul>
<p><span class="font7">Gradient descent is one of the optimization algorithms for the problem of unconstrained minimization. The problem is to minimize </span><span class="font7" style="font-style:italic;">f :I → W,I </span><span class="font3" style="font-style:italic;">⊂</span><span class="font7" style="font-style:italic;"> W</span><span class="font5" style="font-style:italic;"><sup>n</sup></span><span class="font7">. The iteration renews </span><span class="font7" style="font-style:italic;">x<sub>k</sub></span><span class="font7"> so that in the end of iterations, there is an approximation of the above minimization problem. To find the local minimum of a function using gradient descent, one of the important steps is to determine the descent direction, </span><span class="font7" style="font-style:italic;">d</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7">, as the negative of the gradient </span><span class="font7" style="font-style:italic;">g</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7"> = </span><span class="font7" style="font-style:italic;">Vf(x)</span><span class="font7"> of the objective function </span><span class="font7" style="font-style:italic;">f (x)</span><span class="font7"> at </span><span class="font7" style="font-style:italic;">k<sup>tfl</sup></span><span class="font7"> iteration point. Next, select the step size </span><span class="font7" style="font-style:italic;">a</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7"> by using line search, either in exact or inexact.</span></p>
<p><span class="font7">For more details about this method, the following is gradient descent algorithm.</span></p>
<p><span class="font7">Step 0. Given starting point x</span><span class="font5"><sub>0</sub></span><span class="font7">, </span><span class="font5" style="font-style:italic;">k — 0.</span></p>
<p><span class="font7">Step 1. Select the descent direction </span><span class="font7" style="font-style:italic;">d </span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7"> — </span><span class="font7" style="font-style:italic;">-g</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7">. If </span><span class="font7" style="font-style:italic;">d <sub>k</sub></span><span class="font7"> — 0, then the iteration stops.</span></p>
<p><span class="font7">Step 2. Select step size </span><span class="font7" style="font-style:italic;">a</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7"> using line search method (either exact or not exact).</span></p>
<p><span class="font7">Step 3. Renew the point </span><span class="font5" style="font-style:italic;">x<sub>k</sub></span><span class="font5"><sub>+1</sub> </span><span class="font7" style="font-style:italic;">— x<sub>k</sub> + a</span><span class="font5" style="font-style:italic;"><sub>k</sub>d<sub>k</sub></span><span class="font7" style="font-style:italic;">,k</span><span class="font7"> — </span><span class="font7" style="font-style:italic;">k</span><span class="font7"> + 1. Return to step 1.</span></p>
<p><span class="font7">The iteration stops if the iteration termination criterion has been reached. for example if the </span><span class="font7" style="font-style:italic;">k</span><span class="font7"> -th point is close enough to the minimum point of the destination function.</span></p>

<h3><a name="bookmark10"></a><span class="font7" style="font-weight:bold;"><a name="bookmark11"></a>2.3. &nbsp;&nbsp;&nbsp;Line Search Method</span></h3></li></ul>
<p><span class="font7">To determine the step size, </span><span class="font7" style="font-style:italic;">a</span><span class="font5" style="font-style:italic;"><sub>k</sub></span><span class="font7">, there are many line search methods that can be used, either in exact or inexact. <h3><a name="bookmark27"></a><span class="font7" style="font-weight:bold;"><a name="bookmark28"></a>4. CONCLUSIONS</span></h3>
<p><span class="font7">From the discussion above, it can be concluded that modified Armijo works very well on the gradient descent, not only in terms of the number of iterations required to achieve a stopping criterion, but also of computation time on the test functions tested. The parameter value </span><span class="font7" style="font-style:italic;">μ,</span><span class="font7"> affecting its numerical performance, ie with the increasing value of this parameter, then the required iteration also becomes less. Similarly, the effect on program computing time.</span></p>
<p><span class="font7">The Armijo modification works very poorly on the conjugate gradient so that increase of parameter value </span><span class="font7" style="font-style:italic;">μ</span><span class="font7"> cannot be seen affecting the number of iterations required.</span></p>
<p><span class="font7">The two variable test functions have not been able to answer exactly the comparison of the effectiveness of the application of modified Armijo on gradient descent and conjugate gradient. The larger dimension test functions respond to the comparison of the effective application of modified Armijo to the gradient descent and conjugate gradient. The modified Armijo is more effectively applied to gradient descent than conjugate gradient hybrid Gilbert-Nocedal.</span></p>
