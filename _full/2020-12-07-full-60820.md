<h1><a name="bookmark0"></a><span class="font15" style="font-weight:bold;"><a name="bookmark1"></a>Perancangan Sistem Penyedia File Sharing dengan Enkripsi</span><br><br><span class="font15" style="font-weight:bold;"><a name="bookmark2"></a>URL menggunakan Algoritma Rijndael</span></h1>
<p><span class="font13">Adi Bhaskara<sup>1</sup>, Dewa Made Wiharta<sup>2</sup>, Oka Saputra<sup>3</sup></span></p>
<p><span class="font11" style="font-style:italic;">[Submission: 05-11-2020, Accepted: 03-12-2020]</span></p>
<p><span class="font11" style="font-weight:bold;font-style:italic;">Abstract</span><span class="font11" style="font-weight:bold;">—Information is growing very rapidly makes information widely spread and can be disseminated easily and quickly. Generally, a website uses website-based programming with GET method where GET method has a weakness, system data can be seen in access directory. It may cause data in directory to be retrieved by unauthorized people. Based on that problem, researchers offered a solution by creating a file sharing system to share data safely. On system there are applications for uploaders, applications for download for downloaders and admins who regulate number of both application users. File Sharing system has been implemented can storage file directories on website by encrypting files with Rijndael Algorithm. Object in this research using URLs. Testing security level of URL folder that is taken without encryption algorithm Rijndael studied found that directory visible and it can be downloaded files without using an application and without registration. In addition, testing length of URL address shows that increase in length of URL does not affect length of time for encryption and decryption process because in addition to length of URL address, many factors affect, such as traffic on network or server usage load, etc. so that it gets fluctuating results.</span></p>
<p><span class="font11" style="font-weight:bold;font-style:italic;">Keywords</span><span class="font11" style="font-weight:bold;">— GET method, encryption, URL, Rijndael algorithm, authentication, download, upload, ciphertext</span></p>
<p><span class="font11" style="font-weight:bold;font-style:italic;">Intisari</span><span class="font11" style="font-weight:bold;">— Informasi yang berkembang sangat pesat membuat Informasi tersebar secara luas dan dapat disebarkan secara mudah dan cepat. Umumnya sebuah </span><span class="font11" style="font-weight:bold;font-style:italic;">website</span><span class="font11" style="font-weight:bold;"> menggunakan pemrograman berbasis </span><span class="font11" style="font-weight:bold;font-style:italic;">website</span><span class="font11" style="font-weight:bold;"> dengan metode GET dimana metode GET memiliki kelemahan data sistem dapat dilihat di direktori akses. Hal ini menyebabkan data yang ada pada direktori tersebut dapat diambil oleh orang yang tidak berwenang. Berdasarkan masalah tersebut peneliti menawarkan solusi dengan membuat sistem file sharing layanan untuk membagikan data dan melakukan unduh data secara aman. Pada sistem terdapat aplikasi untuk unggah untuk </span><span class="font11" style="font-weight:bold;font-style:italic;">uploader</span><span class="font11" style="font-weight:bold;">, aplikasi untuk unduh untuk </span><span class="font11" style="font-weight:bold;font-style:italic;">downloader</span><span class="font11" style="font-weight:bold;"> dan admin yang bertugas mengatur jumlah kedua pengguna aplikasi. Sistem penyedia File Sharing yang telah diimplementasikan dapat mengamankan direktori file pada website dengan melakukan enkripsi pada direktori file yang dalam penelitian ini yaitu URL dengan menggunakan Algoritma Rijndael. Pengujian tingkat keamanan URL folder yang dibagikan tanpa enkripsi algoritma Rijndael didapatkan bahwa terlihat direktori folder yang dapat dilakukan unduh file tanpa menggunakan aplikasi dan tanpa registrasi. Selain itu, pengujian panjang alamat URL didapatkan</span></p>
<p><span class="font11" style="font-weight:bold;">hasil dengan kenaikan panjang alamat URL tidak mempengaruhi lama waktu proses enkripsi dan dekripsi karena selain panjang alamat URL banyak faktor yang mempengaruhi seperti trafik pada jaringan atau beban penggunaan server, dan lain-lain sehingga mendapatkan hasil yang fluktuatif.</span></p>
<p><span class="font11" style="font-weight:bold;font-style:italic;">Kata Kunci</span><span class="font11" style="font-weight:bold;">— metode GET, enkripsi, URL, algoritma Rijndael, autentikasi, </span><span class="font11" style="font-weight:bold;font-style:italic;">download, upload</span><span class="font11" style="font-weight:bold;">, cipherteks</span></p>
<ul style="list-style:none;"><li>
<h3><a name="bookmark3"></a><span class="font12"><a name="bookmark4"></a>I.</span><span class="font12" style="font-variant:small-caps;"> &nbsp;&nbsp;&nbsp;pendahuluan</span></h3></li></ul>
<p><span class="font12">Pertukaran informasi pada era digital ini memungkinkan semua orang dapat saling bertukar informasi baik berupa data maupun teks dengan cepat dan mudah melalui suatu website. Umumnya sebuah website menggunakan pemrograman berbasis </span><span class="font12" style="font-style:italic;">website</span><span class="font12"> dengan metode GET. metode GET memiliki kelemahan yaitu direktori akses terlihat pada </span><span class="font12" style="font-style:italic;">address bar browser</span><span class="font12"> [1]. Apabila direktori file pada website diketahui maka seluruh data yang ada pada direktori tersebut dapat diambil oleh orang yang tidak berwenang. Pengambilan data oleh orang yang tidak berwenang sangat merugikan terutama bagi website penyedia data berbayar.</span></p>
<p><span class="font12">Salah satu mekanisme untuk meningkatkan keamanan adalah dengan menggunakan teknologi enkripsi pada kriptografi. Kriptografi merupakan teknik pengamanan data yang berguna melindungi kerahasiaan dan keaslian data, serta informasi atau data [2]. Enkripsi diterapkan pada URL sehingga teks pada URL tidak dapat dimengerti orang. URL </span><span class="font12" style="font-style:italic;">(Uniform Resource Locator)</span><span class="font12"> saat ini merupakan salah satu media untuk membagikan data dengan mudah dan cepat terutama pada website-website </span><span class="font12" style="font-style:italic;">file hosting</span><span class="font12">.</span></p>
<p><span class="font12">Penggunaan algoritma simetris dipilih pada penelitian ini karena dibandingkan menggunakan asimetris key dengan kunci yang berbeda untuk enkripsi dan dekripsi, mereka lebih memilih simetris key karena lebih simpel [3]. Selain itu, proses enkripsi algoritma asimetris 1000 kali lebih lambat dibandingkan enkripsi karena memerlukan tenaga komputasional yang lebih dibanding enkripsi algoritma simetris [3]. Terkait penelitian-penelitian tersebut, peneliti melakukan hal yang sama yaitu memilih menggunakan kunci simetris pada sistem file sharing. Namun, algoritma simetris memiliki kelemahan pada saat membagikan kunci simetris dari pemberi ke penerima [4]. Solusi mengamankan pendistribusian kunci yang bersifat privat pada kunci simetris pada penelitian ini di lakukan dengan melakukan enkripsi oleh sistem dengan kunci yang disimpan pada </span><span class="font12" style="font-style:italic;">database</span><span class="font12">.</span></p>
<p><span class="font12">Algoritma kriptografi yang digunakan untuk proses enkripsi pada penelitian ini yaitu Algoritma Rijndael. Alasan pada penelitian ini menggunakan Algoritma Rijndael yaitu keseluruhan kunci dari Rijndael cukup untuk melindungi informasi yang ada didalamnya. Belum ada yang bisa membuktikan keberhasilan serangan melawan Algoritma p-ISSN:1693 – 2951; e-ISSN: </span><span class="font4">2503-2372</span></p><img src="https://jurnal.harianregional.com/media/60820-1.png" alt="" style="width:73pt;height:45pt;">
<p><span class="font12">Rijndael [5]. Penelitian lain juga menjelaskan algoritma AES lebih baik dibanding algoritma DES dalam waktu compile [6]. Berdasarkan waktu yang dibutuhkan untuk memecahkan algoritma kriptografi, semakin lama waktu yang dibutuhkan untuk memecahkan algoritma tersebut maka semakin baik keamanan algoritma tersebut. Sesuai hasil percobaan sebanyak 4 kali pada teks yang berbeda, algoritma Algoritma Rijndael membutuhkan waktu yang paling lama untuk dipecahkan dibanding algoritma DES, E-DES, T-DES, RSA, dan Blowfish [7]. Selain itu, Algoritma AES memiliki performa yang lebih baik dari segi </span><span class="font12" style="font-style:italic;">throughput</span><span class="font12">, waktu enkripsi dan waktu dekripsi dibanding algoritma DES, RSA,dan Blowfish [4]. Bukan hanya itu, Performa AES lebih unggul dibanding algoritma DES, 3DES (112 bit key), 3DES (168 bit key), dan Blowfish sehingga cocok diterapkan pada MANET [8].</span></p>
<p><span class="font12">Penelitian ini mengusulkan suatu metode keamanan file dengan enkripsi URL yang diimplementasikan pada sistem </span><span class="font12" style="font-style:italic;">file sharing</span><span class="font12">. Berdasarkan pada penelitian-penelitian yang terkait yaitu penelitian-penelitian sebelumnya yaitu hanya melakukan enkripsi Rijndael pada URL [9], [10] sedangkan pada penelitian ini dirancang suatu sistem penyedia </span><span class="font12" style="font-style:italic;">file sharing</span><span class="font12"> dengan enkripsi URL menggunakan algoritma Rijndael. Penelitian yang sejenis sebelumnya juga yaitu Perancangan Sistem Aplikasi </span><span class="font12" style="font-style:italic;">Download Manage</span><span class="font12">r Dengan Enkripsi Pada URL Menggunakan Algoritma Blowfish tidak spesifik menjelaskan sistem </span><span class="font12" style="font-style:italic;">file sharing</span><span class="font12"> dimana terdapat aplikasi </span><span class="font12" style="font-style:italic;">upload</span><span class="font12">, aplikasi </span><span class="font12" style="font-style:italic;">download</span><span class="font12">, dan website untuk mengatur pengguna pada kedua aplikasi dan hanya menekankan pada aplikasi </span><span class="font12" style="font-style:italic;">download</span><span class="font12">. Selain itu, kebaharuan terdapat pada sistem </span><span class="font12" style="font-style:italic;">file sharing</span><span class="font12"> yang diimplementasikan menggunakan algoritma Rijndael dimana penelitian sebelumnya proses enkripsi URL menggunakan Algoritma Blowfish [11]. Pada sistem file sharing terdapat 3 bagian utama yaitu aplikasi untuk </span><span class="font12" style="font-style:italic;">upload file</span><span class="font12">, aplikasi untuk </span><span class="font12" style="font-style:italic;">download file</span><span class="font12">, dan halaman admin. Aplikasi </span><span class="font12" style="font-style:italic;">upload </span><span class="font12">digunakan oleh penyedia file untuk mengunggah file yang ingin dibagikan. Orang yang memiliki kewenangan untuk mendapatkan data yang dibagikan menggunakan aplikasi </span><span class="font12" style="font-style:italic;">download</span><span class="font12"> untuk mengunduh file. Sedangkan admin bertugas untuk mengatur jumlah pengguna sebagai </span><span class="font12" style="font-style:italic;">uploader</span><span class="font12"> dan </span><span class="font12" style="font-style:italic;">downloader</span><span class="font12"> file. Pengidentifikasian antara </span><span class="font12" style="font-style:italic;">uploader</span><span class="font12"> dan </span><span class="font12" style="font-style:italic;">downloader</span><span class="font12"> dilakukan dengan proses autentikasi pada masing-masing aplikasi.</span></p>
<ul style="list-style:none;"><li>
<h3><a name="bookmark5"></a><span class="font12"><a name="bookmark6"></a>II.</span><span class="font12" style="font-variant:small-caps;"> &nbsp;&nbsp;&nbsp;Tinjauan Pustaka</span></h3></li></ul>
<p><span class="font12">Pada penelitian digunakan tipe sistem </span><span class="font12" style="font-style:italic;">file sharing centralized</span><span class="font12">. Penelitian ini menggunakan sistem </span><span class="font12" style="font-style:italic;">file sharing centralized</span><span class="font12"> dibanding </span><span class="font12" style="font-style:italic;">decentralized</span><span class="font12"> karena </span><span class="font12" style="font-style:italic;">decentralized </span><span class="font12">memiliki banyak kelemahan keamanan contohnya </span><span class="font12" style="font-style:italic;">file sharing </span><span class="font12">berbasis </span><span class="font12" style="font-style:italic;">cloud</span><span class="font12"> sedangkan </span><span class="font12" style="font-style:italic;">centralized</span><span class="font12"> dimana sistem dikontrol secara terpusat. Berdasarkan penelitian yang ada </span><span class="font12" style="font-style:italic;">cloud computing</span><span class="font12"> banyak memberikan user kenyamanan </span><span class="font12" style="font-style:italic;">service</span><span class="font12">, tetapi disisi lain banyak masalah kritis pada keamanan dan reliabilitas sistem </span><span class="font12" style="font-style:italic;">cloud</span><span class="font12">. Kasus yang sering terjadi yaitu pengambilan data dan kebocoran </span><span class="font12" style="font-style:italic;">user data</span><span class="font12"> [12]. selain itu, sistem </span><span class="font12" style="font-style:italic;">file sharing</span><span class="font12"> dengan menggunakan </span><span class="font12" style="font-style:italic;">cloud server </span><span class="font12">memiliki kelemahan yaitu seseorang dengan akses ke server bisa memodifikasi data tanpa ijin klien [13]. Penelitian ini juga menggunakan teknik keamanan kriptografi dimana</span></p>
<p><span class="font12">kriptografi melindungi konten selama transmisi tanpa pengecekan tambahan setelah di dekripsi [14]. Selain itu kriptografi menyediakan banyak keamanan layanan, kenyamanan, integritas, dan autentikasi [15]. Implementasi algoritma enkripsi yang digunakan pada penelitian ini yaitu algoritma Rijndael dengan kunci 128 bit untuk mengamankan URL dari kelemahan metode GET. algoritma Rijndael sangat kompetitif dengan algoritma Blowfish dilihat dari waktu proses enkripsi dan dekripsi [7]. Namun pada penelitian ini diplih algoritma Rijndael karena evaluasi performa yang dilakukan menggunakan </span><span class="font12" style="font-style:italic;">tool Pycrypto package</span><span class="font12"> pada bahasa pemrograman phyton mendapatkan hasil algoritma Rijndael lebih cepat 200 sampai 300 ms dibandingkan Blowfish [16].</span></p>
<p><span class="font12">Sebelum masuk ke proses enkripsi yang dijelaskan pada gambar sekian, Hal pertama yang dilakukan yaitu melakukan proses key schedule untuk mendapatkan round key yang digunakan pada proses add round key. Penjelasan operasi key schedule yaitu sebagai berikut :</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">• &nbsp;&nbsp;</span><span class="font12">Operasi </span><span class="font12" style="font-style:italic;">Rotate</span><span class="font12">, yaitu operasi perputaran 8 bit pada</span></p></li></ul>
<p><span class="font12">32 bit dari kunci.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">• &nbsp;&nbsp;</span><span class="font12">Operasi </span><span class="font12" style="font-style:italic;">SubBytes</span><span class="font12">, pada operasi ini 8 bit dari </span><span class="font12" style="font-style:italic;">subkey</span></p></li></ul>
<p><span class="font12">disubstitusikan dengan nilai dari S-Box.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">•</span><span class="font12"> &nbsp;&nbsp;&nbsp;Operasi Rcon, operasi ini dapat diterjemahkan sebagai operasi pangkat 2 nilai tertentu dari </span><span class="font12" style="font-style:italic;">user</span><span class="font12">. Operasi ini menggunakan nilai-nilai dalam </span><span class="font12" style="font-style:italic;">Galois field</span><span class="font12">. Nilai-nilai dari Rcon kemudian akan di-XOR dengan hasil operasi </span><span class="font12" style="font-style:italic;">SubBytes</span><span class="font12">.</span></p></li>
<li>

<p><span class="font12">Proses </span><span class="font12" style="font-style:italic;">Shift Rows</span><span class="font12"> akan beroperasi pada tiap baris dari tabel </span><span class="font12" style="font-style:italic;">state</span><span class="font12">. Proses ini akan bekerja dengan cara memutar </span><span class="font12" style="font-style:italic;">bytebyte</span><span class="font12"> pada 3 baris terakhir (baris 1, 2, dan 3) dengan jumlah perputaran yang berbeda-beda. <p><span class="font12">Proses </span><span class="font12" style="font-style:italic;">Mix Columns</span><span class="font12"> akan beroperasi pada tiap kolom dari tabel </span><span class="font12" style="font-style:italic;">state</span><span class="font12">. Kebalikkan dari </span><span class="font12" style="font-style:italic;">polynomial</span><span class="font12"> ini adalah c( x ) = 11x^3+ 13x^2+9x+14. Sedangkan dari 47 kasus uji aplikasi untuk user didapatkan 47 nilai kesimpulan valid dan 25 kasus uji pada website didapatkan 25 nilai kesimpulan valid. Hasil input dari aplikasi untuk user, aplikasi untuk </span><span class="font12" style="font-style:italic;">uploader</span><span class="font12"> dan </span><span class="font12" style="font-style:italic;">website</span><span class="font12"> sesuai dengan output yang diharapkan.</span></p>
<ul style="list-style:none;"><li>
<h3><a name="bookmark25"></a><span class="font12"><a name="bookmark26"></a>IV.</span><span class="font12" style="font-variant:small-caps;"> &nbsp;&nbsp;&nbsp;Kesimpulan</span></h3></li></ul>
<p><span class="font12">Kesimpulan yang dapat diambil penelitian yang telah dilakukan yaitu sebagai berikut :</span></p>
<ul style="list-style:none;"><li>
<p><span class="font12">1. &nbsp;&nbsp;&nbsp;Sistem penyedia </span><span class="font12" style="font-style:italic;">File Sharing</span><span class="font12"> yang telah diimplementasikan dapat mengamankan direktori file pada website dengan melakukan enkripsi pada direktori file yang dalam penelitian ini yaitu URL dengan menggunakan Algoritma Rijndael. Bukan hanya itu, enkripsi yang dilakukan pada URL menggunakan Algoritma Rijndael yang dimana </span><span class="font12" style="font-style:italic;">key </span><span class="font12">dari URL terenkripsi tersimpan pada </span><span class="font12" style="font-style:italic;">database </span><span class="font12">sehingga orang yang tidak berkepentingan tidak dapat menyebarkan URL asli. Selain itu terdapat autentikasi pada masing-masing aplikasi file sharing yaitu aplikasi </span><span class="font12" style="font-style:italic;">uploader</span><span class="font12"> dan aplikasi </span><span class="font12" style="font-style:italic;">downloader </span><span class="font12">sehingga hanya orang yang berwenang yang dapat menggunakan aplikasi.</span></p></li>
<li>
<p><span class="font12">2. &nbsp;&nbsp;&nbsp;Berdasarkan pengujian tingkat keamanan URL folder yang dibagikan tanpa enkripsi algoritma Rijndael didapatkan bahwa terlihat direktori folder yang dapat dilakukan </span><span class="font12" style="font-style:italic;">download file</span><span class="font12"> tanpa menggunakan aplikasi dan tanpa registrasi. Hal tersebut memiliki resiko data dapat didownload oleh orang yang tidak berwenang dan URL dapat disebarluaskan.</span></p></li>
<li>
<p><span class="font12">3. &nbsp;&nbsp;&nbsp;Pengujian Panjang Alamat URL didapatkan hasil dengan kenaikan panjang alamat URL tidak mempengaruhi lama waktu proses enkripsi dan dekripsi karena selain panjang alamat URL banyak faktor yang mempengaruhi seperti trafik pada jaringan atau beban penggunaan server, dan lain-lain sehingga mendapatkan hasil yang fluktuatif.</span></p></li></ul>
