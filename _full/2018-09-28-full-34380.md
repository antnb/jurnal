<h2><a name="bookmark0"></a><span class="font11" style="font-weight:bold;"><a name="bookmark1"></a>Implementasi Algoritma </span><span class="font11" style="font-weight:bold;font-style:italic;">FP-Growth</span><span class="font11" style="font-weight:bold;"> dengan </span><span class="font11" style="font-weight:bold;font-style:italic;">Closure Table </span><span class="font11" style="font-weight:bold;">untuk Penemuan </span><span class="font11" style="font-weight:bold;font-style:italic;">Frequent Itemset</span><span class="font11" style="font-weight:bold;"> pada Keranjang Belanja</span></h2>
<p><span class="font9">I Gusti Agung Indrawan<sup>1</sup>, Made Sudarma<sup>2</sup>, Lie Jasa<sup>3</sup></span></p>
<p><span class="font7" style="font-weight:bold;font-style:italic;">Abstract</span><span class="font7" style="font-weight:bold;">— FP-growth algorithm is a data mining algorithm used to find frequent itemset in market basket data. Frequent itemset is a group of items that are often purchased together in one market basket. Analysis of frequent itemset will result in association rules. FP-growth algorithm finds frequent itemset by compressing market basket data into a tree structure called FP-tree. FP-tree is then analyzed to extract frequent itemset. Market basket data is always increasing for every transaction that occurs, so the process of mining frequent itemset will re-create FP-tree from scratch repeatedly every time FP-growth algorithm executed. In order for FP-tree not to be re-created from the beginning every time FP-growth algorithm executed, FP-tree needs to be stored on storage media using format suitable for tree structure. This research stores FP-tree to table in database using closure table structure. The structure of closure table has several advantages that are suitable for storing tree structure. The results obtained from storing FP-tree to table in database using closure table structure is FP-tree that has been stored in database can be analyzed repeatedly without needing to be recreated from scratch, and only updated when market basket data increases.</span></p>
<p><span class="font7" style="font-weight:bold;font-style:italic;">Keywords</span><span class="font7" style="font-weight:bold;">— </span><span class="font7" style="font-weight:bold;font-style:italic;">closure table, data mining</span><span class="font7" style="font-weight:bold;">, FP-</span><span class="font7" style="font-weight:bold;font-style:italic;">growth</span><span class="font7" style="font-weight:bold;">, FP-</span><span class="font7" style="font-weight:bold;font-style:italic;">tree</span><span class="font7" style="font-weight:bold;">, </span><span class="font7" style="font-weight:bold;font-style:italic;">frequent itemset</span><span class="font7" style="font-weight:bold;">, </span><span class="font7" style="font-weight:bold;font-style:italic;">market basket</span><span class="font7" style="font-weight:bold;">.</span></p>
<p><span class="font7" style="font-weight:bold;font-style:italic;">Intisari</span><span class="font7" style="font-weight:bold;">— Algoritma </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-growth</span><span class="font7" style="font-weight:bold;"> adalah algoritma </span><span class="font7" style="font-weight:bold;font-style:italic;">data mining </span><span class="font7" style="font-weight:bold;">yang digunakan untuk menemukan </span><span class="font7" style="font-weight:bold;font-style:italic;">frequent itemset</span><span class="font7" style="font-weight:bold;"> pada data keranjang belanja. </span><span class="font7" style="font-weight:bold;font-style:italic;">Frequent itemset</span><span class="font7" style="font-weight:bold;"> adalah kelompok barang yang sering dibeli bersamaan dalam satu keranjang belanja. Analisa </span><span class="font7" style="font-weight:bold;font-style:italic;">frequent itemset</span><span class="font7" style="font-weight:bold;"> menghasilkan aturan asosiasi. Algoritma </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-growth</span><span class="font7" style="font-weight:bold;"> menemukan </span><span class="font7" style="font-weight:bold;font-style:italic;">frequent itemset</span><span class="font7" style="font-weight:bold;"> dengan mengkompresi data keranjang belanja ke struktur data </span><span class="font7" style="font-weight:bold;font-style:italic;">tree</span><span class="font7" style="font-weight:bold;"> yang disebut </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-tree. FP-tree</span><span class="font7" style="font-weight:bold;"> kemudian dianalisa untuk mengekstrak </span><span class="font7" style="font-weight:bold;font-style:italic;">frequent itemset</span><span class="font7" style="font-weight:bold;">. Data keranjang belanja selalu bertambah jumlahnya untuk setiap transaksi yang terjadi, sehingga proses penemuan </span><span class="font7" style="font-weight:bold;font-style:italic;">frequent itemset</span><span class="font7" style="font-weight:bold;"> akan membuat </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-tree</span><span class="font7" style="font-weight:bold;"> dari awal secara berulang-ulang setiap kali algoritma </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-growth</span><span class="font7" style="font-weight:bold;"> dijalankan. Agar </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-tree</span><span class="font7" style="font-weight:bold;"> tidak perlu dibuat ulang dari awal setiap kali algoritma </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-growth</span><span class="font7" style="font-weight:bold;"> dijalankan, maka </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-tree</span><span class="font7" style="font-weight:bold;"> perlu disimpan pada media penyimpanan menggunakan format yang sesuai dengan struktur data </span><span class="font7" style="font-weight:bold;font-style:italic;">tree</span><span class="font7" style="font-weight:bold;">. Penelitian ini menyimpan </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-tree</span><span class="font7" style="font-weight:bold;"> ke tabel pada </span><span class="font7" style="font-weight:bold;font-style:italic;">database</span><span class="font7" style="font-weight:bold;"> dengan struktur </span><span class="font7" style="font-weight:bold;font-style:italic;">closure table</span><span class="font7" style="font-weight:bold;">. Struktur </span><span class="font7" style="font-weight:bold;font-style:italic;">closure table </span><span class="font7" style="font-weight:bold;">memiliki beberapa keunggulan sehingga cocok digunakan untuk menyimpan struktur data </span><span class="font7" style="font-weight:bold;font-style:italic;">tree</span><span class="font7" style="font-weight:bold;">. Hasil yang didapatkan dari penyimpanan </span><span class="font7" style="font-weight:bold;font-style:italic;">FP-tree</span><span class="font7" style="font-weight:bold;"> ke tabel pada </span><span class="font7" style="font-weight:bold;font-style:italic;">database</span><span class="font7" style="font-weight:bold;"> menggunakan struktur </span><span class="font7" style="font-weight:bold;font-style:italic;">closure table adalah FP-tree</span><span class="font7" style="font-weight:bold;"> yang telah tersimpan pada </span><span class="font7" style="font-weight:bold;font-style:italic;">database</span><span class="font7" style="font-weight:bold;"> dapat dianalisa berulang kali tanpa perlu dibuat dari awal, dan hanya di-</span><span class="font7" style="font-weight:bold;font-style:italic;">update</span><span class="font7" style="font-weight:bold;"> ketika data keranjang belanja bertambah.</span></p>
<p><span class="font7" style="font-weight:bold;font-style:italic;">Kata Kunci</span><span class="font7" style="font-weight:bold;">— </span><span class="font7" style="font-weight:bold;font-style:italic;">closure table, data mining</span><span class="font7" style="font-weight:bold;">, FP-</span><span class="font7" style="font-weight:bold;font-style:italic;">growth</span><span class="font7" style="font-weight:bold;">, FP-</span><span class="font7" style="font-weight:bold;font-style:italic;">tree, frequent itemset,</span><span class="font7" style="font-weight:bold;"> keranjang belanja.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font7">I.</span><span class="font8"> &nbsp;&nbsp;&nbsp;PENDAHULUAN</span></p></li></ul>
<p><span class="font8">Usaha </span><span class="font8" style="font-style:italic;">retail</span><span class="font8"> yang menggunakan sistem </span><span class="font8" style="font-style:italic;">point-of-sales </span><span class="font8">menyimpan data transaksi penjualan barang dalam jumlah besar. Data transaksi penjualan barang per pelanggan disebut data keranjang belanja. Data transaksi yang tersimpan dalam jumlah besar ini berpotensi memiliki </span><span class="font8" style="font-style:italic;">knowledge</span><span class="font8"> yang belum diketahui. Sebagai contoh adalah </span><span class="font8" style="font-style:italic;">knowledge</span><span class="font8"> tentang barang yang sering dibeli secara bersamaan dalam satu transaksi, yang dalam istilah </span><span class="font8" style="font-style:italic;">data mining</span><span class="font8"> disebut </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8">. </span><span class="font8" style="font-style:italic;">Knowledge</span><span class="font8"> tersebut dapat digunakan oleh </span><span class="font8" style="font-style:italic;">retailer</span><span class="font8"> untuk meningkatkan penjualan, misal dengan membuat promosi bundel barang-barang tertentu dan meletakkan barang yang sering dibeli secara berdekatan pada rak penjualan. Penelitian untuk menemukan </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> pada keranjang belanja sebelumnya dilakukan oleh [1].</span></p>
<p><span class="font8">Algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> menyimpan data keranjang belanja ke struktur data </span><span class="font8" style="font-style:italic;">tree</span><span class="font8"> yang disebut FP-</span><span class="font8" style="font-style:italic;">tree</span><span class="font8">. Algorima FP</span><span class="font8" style="font-style:italic;">-growth</span><span class="font8"> yang dijabarkan oleh [2] dan [3] membangun dan menyimpan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">random access memory</span><span class="font8"> (RAM). Kelemahan penyimpanan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> pada RAM adalah </span><span class="font8" style="font-style:italic;">FP-tree </span><span class="font8">akan terhapus ketika program dihentikan atau komputer dimatikan. Solusi dari masalah tersebut adalah menyimpan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> yang telah di-</span><span class="font8" style="font-style:italic;">generate</span><span class="font8"> ke media penyimpanan data pada komputer. Struktur </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> adalah solusi yang elegan untuk menyimpan hierarki </span><span class="font8" style="font-style:italic;">tree</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">database</span><span class="font8">. </span><span class="font8" style="font-style:italic;">Closure table</span><span class="font8"> menyimpan seluruh </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">tree</span><span class="font8"> untuk </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> tertentu, tidak hanya </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> dengan relasi </span><span class="font8" style="font-style:italic;">parent-child</span><span class="font8"> [4]. Keunggulan struktur </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> adalah </span><span class="font8" style="font-style:italic;">query</span><span class="font8"> untuk menemukan seluruh </span><span class="font8" style="font-style:italic;">child node</span><span class="font8"> untuk </span><span class="font8" style="font-style:italic;">parent node</span><span class="font8"> tertentu hanya memerlukan satu kali </span><span class="font8" style="font-style:italic;">query</span><span class="font8">.</span></p>
<p><span class="font8">Penggunaan </span><span class="font8" style="font-style:italic;">database</span><span class="font8"> untuk menyimpan </span><span class="font8" style="font-style:italic;">FP-tree </span><span class="font8">menggunakan struktur </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> diharapkan menghilangkan keterbatasan algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> dimana ukuran </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> dapat melebihi kapasitas memori utama yang tersedia pada komputer, terutama pada </span><span class="font8" style="font-style:italic;">dataset</span><span class="font8"> keranjang belanja berukuran besar.</span></p>

<p><span class="font7">II.</span><span class="font8"> &nbsp;&nbsp;&nbsp;STUDI LITERATUR</span></p></li></ul>
<ul style="list-style:none;"><li>
<p><span class="font8" style="font-style:italic;">A. &nbsp;&nbsp;&nbsp;Data Mining</span></p></li></ul>
<p><span class="font8">Istilah </span><span class="font8" style="font-style:italic;">data mining</span><span class="font8"> dapat dijelaskan sebagai proses menemukan pengetahuan (</span><span class="font8" style="font-style:italic;">knowledge</span><span class="font8">) atau pola (</span><span class="font8" style="font-style:italic;">pattern</span><span class="font8">) dari data berukuran besar [5]. Seiring dengan meningkatnya komputerisasi kegiatan masyarakat, kapasitas penyimpanan data yang semakin besar dan kemampuan </span><span class="font8" style="font-style:italic;">processing </span><span class="font8">komputer yang semakin cepat, mengakibatkan data yang dihasilkan dan disimpan oleh perorangan maupun organisasi tiap harinya sangatlah besar. Kegiatan sehari-hari seperti transaksi pembelian yang diproses dengan terminal </span><span class="font8" style="font-style:italic;">point-of-</span><span class="font8">p-ISSN:1693 – 2951; e-ISSN: </span><span class="font1">2503-2372</span></p><img src="https://jurnal.harianregional.com/media/34380-1.png" alt="" style="width:59pt;height:39pt;">
<p><span class="font8" style="font-style:italic;">sales</span><span class="font8">, penggunaan situs media sosial untuk melakukan </span><span class="font8" style="font-style:italic;">sharing </span><span class="font8">berita dan foto serta video, adalah contoh kegiatan yang menghasilkan data dalam jumlah besar.</span></p>
<p><span class="font8">Jumlah data yang sangat besar yang tidak dibarengi dengan wawasan (</span><span class="font8" style="font-style:italic;">insight</span><span class="font8">) yang cukup (</span><span class="font8" style="font-style:italic;">knowledge</span><span class="font8">) terhadap data tersebut disebut </span><span class="font8" style="font-style:italic;">data rich but information poor</span><span class="font8">. Hal ini diakibatkan tidak terpenuhinya kebutuhan terhadap alat analisis data yang handal. Situasi inilah yang kemudian memunculkan teknologi </span><span class="font8" style="font-style:italic;">data mining</span><span class="font8">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8" style="font-style:italic;">B. &nbsp;&nbsp;&nbsp;Aturan Asosiasi</span></p></li></ul>
<p><span class="font8" style="font-style:italic;">Frequent patterns</span><span class="font8"> adalah pola yang sering muncul pada </span><span class="font8" style="font-style:italic;">dataset</span><span class="font8">. </span><span class="font8" style="font-style:italic;">Frequent patterns</span><span class="font8"> dapat berupa </span><span class="font8" style="font-style:italic;">itemset</span><span class="font8">, </span><span class="font8" style="font-style:italic;">subsequences </span><span class="font8">atau </span><span class="font8" style="font-style:italic;">substructures</span><span class="font8">. Kumpulan barang yang sering muncul bersamaan pada satu transaksi data, sebagai contoh susu dan roti yang sering dibeli bersamaan pada satu transaksi penjualan barang, disebut </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> [6]. Penemuan </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> pada data transaksi keranjang belanja memungkinkan perusahaan </span><span class="font8" style="font-style:italic;">retail</span><span class="font8"> mengoptimalkan penjualan barang, misal dengan meletakkan barang-barang yang sering dibeli bersamaan pada rak yang berdekatan atau mengadakan promosi barang bundel.</span></p>

<p><span class="font8" style="font-style:italic;">C. &nbsp;&nbsp;&nbsp;Algoritma FP-growth</span></p></li></ul>
<p><span class="font8">Algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> (FP: </span><span class="font8" style="font-style:italic;">frequent-pattern</span><span class="font8">) mengkodekan data menggunakan struktur data yang disebut </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> dan mengekstrak </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> dari </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> tersebut. </span><span class="font8" style="font-style:italic;">FP-tree </span><span class="font8">adalah representasi terkompresi dari data keranjang belanja. </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> dibangun dengan membaca </span><span class="font8" style="font-style:italic;">dataset</span><span class="font8"> transaksi per satu transaksi dan melakukan </span><span class="font8" style="font-style:italic;">mapping item</span><span class="font8"> tiap transaksi tersebut ke sebuah </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> [6]. Dikarenakan setiap transaksi berbeda dapat memiliki beberapa item transaksi yang sama, </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> item transaksi yang sama tersebut akan </span><span class="font8" style="font-style:italic;">overlap</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> di </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8">. Semakin banyak </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> yang </span><span class="font8" style="font-style:italic;">overlap</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8">, semakin besar kompresi data yang bisa dicapai.</span></p>
<p><span class="font8">Penelitian sebelumnya tentang implementasi algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> antara lain:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">1. &nbsp;&nbsp;&nbsp;Referensi [7] membahas tentang optimasi algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> yang dinamakan APFTC (Apriori with </span><span class="font8" style="font-style:italic;">FP-tree and Correlation</span><span class="font8">), dimana </span><span class="font8" style="font-style:italic;">FP-tree mining</span><span class="font8"> untuk</span></p></li></ul>
<p><span class="font8">menemukan </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> menggunakan algoritma </span><span class="font8" style="font-style:italic;">apriori</span><span class="font8">. Algoritma APFTC mengenalkan konsep korelasi untuk meningkatkan kualitas </span><span class="font8" style="font-style:italic;">interestingness</span><span class="font8"> dari aturan asosiasi yang ditemukan.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">2. &nbsp;&nbsp;&nbsp;Referensi [8] membahas tentang algoritma </span><span class="font8" style="font-style:italic;">FP-growth </span><span class="font8">yang dieksekusi menggunakan Apache Hadoop. Apache Hadoop adalah </span><span class="font8" style="font-style:italic;">framework</span><span class="font8"> yang memungkinkan pemrosesan </span><span class="font8" style="font-style:italic;">datasets</span><span class="font8"> berukuran besar secara paralel pada komputer terkluster. Penelitian ini menggunakan implementasi </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> dari Apache Mahout. Apache Mahout adalah </span><span class="font8" style="font-style:italic;">library</span><span class="font8"> untuk mengembangkan aplikasi </span><span class="font8" style="font-style:italic;">machine learning</span><span class="font8">.</span></p></li>
<li>
<p><span class="font8">3. &nbsp;&nbsp;&nbsp;Referensi [9] membahas tentang optimasi algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> yang dinamakan algoritma </span><span class="font8" style="font-style:italic;">painting-growth</span><span class="font8"> dan </span><span class="font8" style="font-style:italic;">n-painting-growth</span><span class="font8">.</span></p></li></ul>
<ul style="list-style:none;"><li>
<p><span class="font8" style="font-style:italic;">D. &nbsp;&nbsp;&nbsp;Closure Table</span></p></li></ul>
<p><span class="font8">Struktur </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> menyimpan seluruh </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">tree </span><span class="font8">untuk </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> tertentu, tidak hanya </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> dengan relasi </span><span class="font8" style="font-style:italic;">parentchild</span><span class="font8"> [4]. Keunggulan struktur </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> adalah kemudahan melakukan </span><span class="font8" style="font-style:italic;">query</span><span class="font8"> seluruh </span><span class="font8" style="font-style:italic;">child node</span><span class="font8"> untuk </span><span class="font8" style="font-style:italic;">parent node</span><span class="font8"> tertentu, sehingga cocok digunakan sebagai struktur tabel untuk menyimpan struktur data </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> yang digunakan algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8">. Struktur </span><span class="font8" style="font-style:italic;">closure table </span><span class="font8">diimplementasikan sebagai tabel tambahan. Tabel tersebut bernama </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8"> dan didefinisikan dengan SQL </span><span class="font8" style="font-style:italic;">query </span><span class="font8">sebagai berikut:</span></p>
<p><span class="font3" style="font-weight:bold;">CREATE TABLE </span><span class="font3">Treepaths (</span></p>
<p><span class="font3">ancestor </span><span class="font3" style="font-weight:bold;">BIGINT UNSIGNED NOT NULL</span><span class="font3">,</span></p>
<p><span class="font3">descendant </span><span class="font3" style="font-weight:bold;">BIGINT UNSIGNED NOT NULL</span><span class="font3">,</span></p>
<p><span class="font3" style="font-weight:bold;">PRIMARY KEY</span><span class="font3">(ancestor, descendant),</span></p>
<p><span class="font3" style="font-weight:bold;">FOREIGN KEY </span><span class="font3">(ancestor) </span><span class="font3" style="font-weight:bold;">REFERENCES </span><span class="font3">trx(id),</span></p>
<p><span class="font3" style="font-weight:bold;">FOREIGN KEY </span><span class="font3">(descendant) </span><span class="font3" style="font-weight:bold;">REFERENCES </span><span class="font3">trx(id)</span></p>
<p><span class="font3">);</span></p>
<p><span class="font8">Tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8"> mereferensi kolom id yang merupakan </span><span class="font8" style="font-style:italic;">primary key</span><span class="font8"> dari tabel trx (transaksi/</span><span class="font8" style="font-style:italic;">market basket</span><span class="font8">). Salah satu ciri struktur </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> adalah setiap </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> mereferensi dirinya sendiri. Setiap </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> juga menyimpan referensi/</span><span class="font8" style="font-style:italic;">path</span><span class="font8"> ke semua </span><span class="font8" style="font-style:italic;">child node</span><span class="font8"> yang ada dibawahnya. Tabel 1 menunjukkan data tabel nama barang, sedangkan tabel 2 menunjukkan data tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8">.</span></p>

<p><span class="font7">III.</span><span class="font8"> &nbsp;&nbsp;&nbsp;METODE</span></p></li></ul>

<p><span class="font8" style="font-style:italic;">A. &nbsp;&nbsp;&nbsp;Alur Implementasi Algoritma FP-growth Menggunakan Closure Table</span></p></li></ul>
<p><span class="font8">Alur implementasi algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> dengan </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> dijelaskan sebagai berikut:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">1. &nbsp;&nbsp;&nbsp;Data keranjang belanja yang berbentuk file teks ditransformasi agar dapat disimpan dalam bentuk tabel pada </span><span class="font8" style="font-style:italic;">database</span><span class="font8">.</span></p></li>
<li>
<p><span class="font8">2. &nbsp;&nbsp;&nbsp;Tentukan </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8"> untuk penemuan </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8">. Item yang tidak memenuhi </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8">, dianggap </span><span class="font8" style="font-style:italic;">infrequent</span><span class="font8"> dan tidak disertakan dalam proses </span><span class="font8" style="font-style:italic;">mining</span><span class="font8">. </span><span class="font8" style="font-style:italic;">Update</span><span class="font8"> kolom is_min_sc pada tabel items dan tabel trx_details dengan nilai </span><span class="font8" style="font-style:italic;">true/false</span><span class="font8"> sesuai dengan kondisi apakah frekuensi kemunculan barang pada </span><span class="font8" style="font-style:italic;">dataset</span><span class="font8"> keranjang belanja memenuhi </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8"> atau tidak.</span></p></li>
<li>
<p><span class="font8">3.</span><span class="font8" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;Scan</span><span class="font8"> tabel trx_detail untuk menentukan semua ID transaksi detail yang valid. ID transaksi detail dianggap valid ketika ID transaksi detail tersebut memiliki ID transaksi master dan frekuensi kemunculan barang pada transaksi detail tersebut pada </span><span class="font8" style="font-style:italic;">dataset</span><span class="font8"> keranjang belanja memenuhi syarat </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8">.</span></p></li>
<li>
<p><span class="font8">4.</span><span class="font8" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;Scan</span><span class="font8"> tabel trx_detail untuk menemukan frekuensi kemunculan item pada keranjang belanja. </span><span class="font8" style="font-style:italic;">Update</span><span class="font8"> jumlah frekuensi kemunculan item ke kolom freq pada tabel item.</span></p></li>
<li>
<p><span class="font8">5. &nbsp;&nbsp;&nbsp;Tahap berikutnya adalah pembuatan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8">. Item transaksi yang telah berurut secara </span><span class="font8" style="font-style:italic;">descending</span><span class="font8">, di-</span></p></li></ul>

<ul style="list-style:none;"><li>
<p><span class="font8">6. &nbsp;&nbsp;&nbsp;Setelah </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> terbentuk, tahap berikutnya adalah ekstraksi </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8">.</span></p></li>

<p><span class="font8" style="font-style:italic;">B. &nbsp;&nbsp;&nbsp;Struktur Tabel di Database</span></p></li></ul>
<p><span class="font8">Struktur data </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> yang dihasilkan oleh algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8"> disimpan sebagai </span><span class="font8" style="font-style:italic;">closure table</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">database</span><span class="font8"> MySQL. Tabel yang menyimpan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">database</span><span class="font8"> MySQL dinamakan tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8"> dan didefinsikan dengan SQL </span><span class="font8" style="font-style:italic;">query</span><span class="font8"> sebagai berikut:</span></p>

<p><span class="font3" style="font-weight:bold;">ALTER TABLE </span><span class="font3">treepaths </span><span class="font3" style="font-weight:bold;">ADD CONSTRAINT </span><span class="font3">fk_treepaths_anc_ref_items_id </span><span class="font3" style="font-weight:bold;">FOREIGN KEY </span><span class="font3">(ancestor) </span><span class="font3" style="font-weight:bold;">REFERENCES </span><span class="font3">items (id_item);</span></p>
<p><span class="font3" style="font-weight:bold;">ALTER TABLE </span><span class="font3">treepaths </span><span class="font3" style="font-weight:bold;">ADD CONSTRAINT </span><span class="font3">fk_treepaths_des_ref_items_id </span><span class="font3" style="font-weight:bold;">FOREIGN KEY </span><span class="font3">(descendant) </span><span class="font3" style="font-weight:bold;">REFERENCES </span><span class="font3">items (id_item);</span></p>

Kolom ancestor menyimpan </span><span class="font8" style="font-style:italic;">id node</span><span class="font8"> yang merupakan </span><span class="font8" style="font-style:italic;">ancestor</span><span class="font8"> dari </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> saat ini, dan kolom descendant menyimpan </span><span class="font8" style="font-style:italic;">id node</span><span class="font8"> yang merupakan </span><span class="font8" style="font-style:italic;">descendant</span><span class="font8"> dari </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> saat ini. <p><span class="font8" style="font-style:italic;">C. &nbsp;&nbsp;&nbsp;Penyimpanan FP-tree ke Closure Table</span></p></li></ul>
<p><span class="font8">Algoritma penyimpanan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> ke </span><span class="font8" style="font-style:italic;">closure table </span><span class="font8">dijelaskan sebagai berikut:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">1. &nbsp;&nbsp;&nbsp;Langkah pertama adalah meng-</span><span class="font8" style="font-style:italic;">insert root node FP-tree </span><span class="font8">ke tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8">. Pada algoritma </span><span class="font8" style="font-style:italic;">FP-growth</span><span class="font8">, </span><span class="font8" style="font-style:italic;">root node </span><span class="font8">dari </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> adalah </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> dengan data </span><span class="font8" style="font-style:italic;">null</span><span class="font8">, dan pada algoritma ini diimplementasikan dengan angka nol sebagai nilai kolom </span><span class="font8" style="font-style:italic;">ancestor</span><span class="font8"> dan </span><span class="font8" style="font-style:italic;">descendant</span><span class="font8">. Kolom </span><span class="font8" style="font-style:italic;">path_length</span><span class="font8"> berisi nilai nol karena baris data ini adalah </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> pertama.</span></p></li>
<li>
<p><span class="font8">2.</span><span class="font8" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;Item</span><span class="font8"> pada keranjang belanja yang memenuhi syarat </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8">, diurutkan secara </span><span class="font8" style="font-style:italic;">descending </span><span class="font8">berdasarkan jumlah kemunculan item tersebut pada </span><span class="font8" style="font-style:italic;">dataset</span><span class="font8"> keranjang belanja.</span></p></li>
<li>
<p><span class="font8" style="font-style:italic;">3. &nbsp;&nbsp;&nbsp;Item</span><span class="font8"> keranjang belanja yang sudah terurut, di-</span><span class="font8" style="font-style:italic;">insert</span><span class="font8"> ke tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8"> berdasarkan </span><span class="font8" style="font-style:italic;">pseudo code</span><span class="font8"> sebagai berikut:</span></p></li></ul>
<p><span class="font3" style="font-weight:bold;">For </span><span class="font3">j = </span><span class="font3" style="font-weight:bold;">0 To </span><span class="font3">jml_item_krnjng_blnj</span></p>
<p><span class="font3" style="font-weight:bold;">If </span><span class="font3">j = </span><span class="font3" style="font-weight:bold;">0 Then</span></p>
<p><span class="font3" style="font-weight:bold;">For </span><span class="font3">k = </span><span class="font3" style="font-weight:bold;">0 To </span><span class="font3">jml_item_krnjng_blnj – </span><span class="font3" style="font-weight:bold;">1 </span><span class="font3">idItem = TrxDetails.Rows(k).idItem ExecQuery(INSERT INTO treepaths(ancestor, descendant, path_length) VALUES(</span><span class="font3" style="font-weight:bold;">0</span><span class="font3">, idItem, k+</span><span class="font3" style="font-weight:bold;">1</span><span class="font3">))</span></p>
<p><span class="font3" style="font-weight:bold;">Next</span></p>
<p><span class="font3" style="font-weight:bold;">Else</span></p>
<p><span class="font3" style="font-weight:bold;">For </span><span class="font3">k = j </span><span class="font3" style="font-weight:bold;">To </span><span class="font3">jml_item_krnjng_blnj idItemAncstr = TrxDetails.Rows(j-</span><span class="font3" style="font-weight:bold;">1</span><span class="font3">).idItem idItemDescndnt = TrxDetails.Rows(k-</span><span class="font3" style="font-weight:bold;">1</span><span class="font3">).idItem ExecQuery(INSERT INTO treepaths(ancestor, descendant, path_length) VALUES(idItemAncstr, idItemDescndnt, k-j))</span></p>
<p><span class="font3" style="font-weight:bold;">Next</span></p>
<p><span class="font3" style="font-weight:bold;">End I</span><span class="font3">f</span></p>

<p><span class="font8" style="font-style:italic;">D. &nbsp;&nbsp;&nbsp;Ekstraksi Frequent Itemset Dari Closure Table</span></p></li></ul>
<p><span class="font8">Algoritma ekstraksi </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> dari tabel </span><span class="font8" style="font-style:italic;">treepaths </span><span class="font8">dijelaskan sebagai berikut:</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">1. &nbsp;&nbsp;&nbsp;Scan tabel items untuk menemukan </span><span class="font8" style="font-style:italic;">item</span><span class="font8"> yang memenuhi kriteria </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8">. Untuk semua item yang memenuhi kriteria tersebut, dapatkan </span><span class="font8" style="font-style:italic;">ancestor</span><span class="font8">-nya menggunakan SQL </span><span class="font8" style="font-style:italic;">query</span><span class="font8"> sebagai berikut:</span></p></li></ul>
<p><span class="font3">Select ancestor FROM treepaths</span></p>
<p><span class="font3">WHERE descendant = idItem</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">2. &nbsp;&nbsp;&nbsp;Hasil dari </span><span class="font8" style="font-style:italic;">query</span><span class="font8"> tersebut adalah </span><span class="font8" style="font-style:italic;">multiple paths</span><span class="font8"> dimana tiap </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> berisi </span><span class="font8" style="font-style:italic;">item</span><span class="font8"> yang dibeli bersamaan dengan </span><span class="font8" style="font-style:italic;">item </span><span class="font8">yang sedang diproses. Tiap </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">multiple paths </span><span class="font8">dipisahkan dengan data </span><span class="font8" style="font-style:italic;">ancestor</span><span class="font8"> = 0 (nol).</span></p></li>
<li>
<p><span class="font8">3. &nbsp;&nbsp;&nbsp;Langkah ketiga adalah memecah </span><span class="font8" style="font-style:italic;">multiple paths</span><span class="font8"> menjadi </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> individual. Pemecahan </span><span class="font8" style="font-style:italic;">multiple paths</span><span class="font8"> kemungkinan</span></p></li></ul>
<p><span class="font8">menghasilkan </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> duplikat yang memiliki urutan/</span><span class="font8" style="font-style:italic;">sequence</span><span class="font8"> id item yang sama. Jumlah kemunculan </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> duplikat yang memiliki urutan/</span><span class="font8" style="font-style:italic;">sequence</span><span class="font8"> id item yang sama tersebut disimpan sebagai </span><span class="font8" style="font-style:italic;">support count path </span><span class="font8">tersebut. </span><span class="font8" style="font-style:italic;">Path</span><span class="font8"> yang duplikat kemudian dihapus sehingga tersisa satu </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> individual yang memiliki </span><span class="font8" style="font-style:italic;">support count </span><span class="font8">yang bersesuaian.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font8">4. &nbsp;&nbsp;&nbsp;Langkah keempat adalah menemukan semua id item unik yang disimpan sebagai </span><span class="font8" style="font-style:italic;">end node</span><span class="font8"> di </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> individual. Semua id item unik yang ditemukan disimpan dalam </span><span class="font8" style="font-style:italic;">list</span><span class="font8">.</span></p></li>
<li>
<p><span class="font8">5. &nbsp;&nbsp;&nbsp;Langkah kelima adalah melakukan iterasi untuk semua id item unik yang ada pada </span><span class="font8" style="font-style:italic;">list</span><span class="font8">, temukan </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> individual yang memiliki </span><span class="font8" style="font-style:italic;">end node</span><span class="font8"> sama dengan id item tersebut. Tambahkan </span><span class="font8" style="font-style:italic;">support count</span><span class="font8"> semua </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> individual yang memiliki </span><span class="font8" style="font-style:italic;">end node</span><span class="font8"> sama dengan id item, sebagai </span><span class="font8" style="font-style:italic;">support count</span><span class="font8"> id item tersebut. Tentukan juga </span><span class="font8" style="font-style:italic;">support count</span><span class="font8"> untuk </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> lain pada </span><span class="font8" style="font-style:italic;">path</span><span class="font8"> individual dengan </span><span class="font8" style="font-style:italic;">end node</span><span class="font8"> sama dengan id item yang sedang diproses.</span></p></li>
<li>
<p><span class="font8">6. &nbsp;&nbsp;&nbsp;Bila </span><span class="font8" style="font-style:italic;">support count</span><span class="font8"> id item </span><span class="font0">≥ </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8">, maka id item tersebut </span><span class="font8" style="font-style:italic;">frequent</span><span class="font8">. Pada titik ini penemuan </span><span class="font8" style="font-style:italic;">frequent</span><span class="font8"> 1-</span><span class="font8" style="font-style:italic;">itemset</span><span class="font8"> sudah selesai.</span></p></li>
<li>
<p><span class="font8">7. &nbsp;&nbsp;&nbsp;Langkah ketujuh adalah penemuan </span><span class="font8" style="font-style:italic;">frequent</span><span class="font8"> 2-</span><span class="font8" style="font-style:italic;">itemset</span><span class="font8">. Pada tahap 5, </span><span class="font8" style="font-style:italic;">support count</span><span class="font8"> untuk </span><span class="font8" style="font-style:italic;">node</span><span class="font8"> lain pada </span><span class="font8" style="font-style:italic;">path </span><span class="font8">individual dengan </span><span class="font8" style="font-style:italic;">end node</span><span class="font8"> sama dengan id item yang sedang diproses sudah dihitung. Bila </span><span class="font8" style="font-style:italic;">support count node </span><span class="font8">lain tersebut </span><span class="font0">≥ </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8">, maka node lain tersebut adalah </span><span class="font8" style="font-style:italic;">frequent</span><span class="font8"> dan anggota </span><span class="font8" style="font-style:italic;">frequent</span><span class="font8"> 2-</span><span class="font8" style="font-style:italic;">itemset </span><span class="font8">dari id item yang sedang diproses.</span></p></li></ul>

<p><span class="font8">1. &nbsp;&nbsp;&nbsp;Langkah pertama adalah melakukan konfigurasi koneksi </span><span class="font8" style="font-style:italic;">database</span><span class="font8"> MySQL.</span></p></li>
<li>
<p><span class="font8">2. &nbsp;&nbsp;&nbsp;Langkah kedua adalah melakukan transformasi data dari data keranjang belanja yang berupa file teks menjadi </span><span class="font8" style="font-style:italic;">record</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">database</span><span class="font8">. Langkah transformasi data ini dilakukan di bagian (1 Data Loader) pada program </span><span class="font8" style="font-style:italic;">FP-miner</span><span class="font8">.</span></p></li>
<li>
<p><span class="font8">3. &nbsp;&nbsp;&nbsp;Langkah ketiga adalah membuat </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> yang akan disimpan pada tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8"> pada </span><span class="font8" style="font-style:italic;">database</span><span class="font8"> MySQL. Langkah pembuatan </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> ini dilakukan di bagian (2 FP-tree Builder) pada program </span><span class="font8" style="font-style:italic;">FP-miner</span><span class="font8">. Penentuan </span><span class="font8" style="font-style:italic;">minimum support count</span><span class="font8"> dilakukan pada langkah ini. </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8"> yang dihasilkan berdasarkan data keranjang belanja alat tulis pada tabel 9 disimpan pada tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8"> dan ditunjukkan pada tabel 10.</span></p></li>
<li>
<p><span class="font8">4. &nbsp;&nbsp;&nbsp;Langkah keempat adalah melakukan ekstraksi </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> dari tabel </span><span class="font8" style="font-style:italic;">treepaths</span><span class="font8">. Langkah ini dilakukan dengan mengklik tombol (Extract FP-tree).</span></p></li>
<li>
<p><span class="font8">5. &nbsp;&nbsp;&nbsp;Langkah kelima adalah pembuatan laporan aturan asosiasi. Pembuatan laporan ini dilakukan dibagian (3 Report Generator) pada program </span><span class="font8" style="font-style:italic;">FP-miner</span><span class="font8">. Penentuan </span><span class="font8" style="font-style:italic;">minimum confidence</span><span class="font8"> aturan asosiasi yang akan dilaporkan ditentukan pada tahap ini. Bila </span><span class="font8" style="font-style:italic;">minimum confidence </span><span class="font8">dikosongkan, maka program akan melaporkan semua aturan asosiasi yang ditemukan. Opsi (Include FI Table) bila dicentang maka pada laporan aturan asosiasi akan ditampilkan data </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> yang ditemukan saat proses ekstraksi </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> dari </span><span class="font8" style="font-style:italic;">FP-tree</span><span class="font8">. Tabel 11 menunjukkan </span><span class="font8" style="font-style:italic;">frequent itemset</span><span class="font8"> untuk data keranjang belanja alat tulis.</span></p></li></ul>
