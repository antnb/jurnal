<p><span class="font3" style="font-weight:bold;font-style:italic;">Abstract</span></p>
<p><span class="font3" style="font-style:italic;">The security of gym member data files is an important aspect of maintaining the privacy and security of personal information stored in the gym management system. In this study, we propose the application of the Advanced Encryption Standard (AES) algorithm as a solution to secure gym member data files. AES is an encryption algorithm known for its high security and has been extensively used in information security applications. In this implementation, each gym member's data files, such as profiles, addresses, and other personal information, will be encrypted using a secret key that only the system knows. With encryption using AES, even if the member's data file is stolen or accessed illegally, the information contained in it remains safe and cannot be read without the correct secret key. The results of the trial found that this application succeeded in encrypting some member data files with .pdf format into the. rda format, then decrypting them back into the original file form without changing the original file size. So, it can be shown that the use of the AES algorithm in securing gym member data files provides a high level of security and protects sensitive information from unauthorized parties.</span></p>
<p><span class="font3" style="font-weight:bold;font-style:italic;">Keywords: </span><span class="font3" style="font-style:italic;">Cryptography, Advanced Encryption Standard (AES), File Security, Gym Member Data</span></p>

<h1><a name="bookmark0"></a><span class="font3" style="font-weight:bold;"><a name="bookmark1"></a>1. &nbsp;&nbsp;&nbsp;Pendahuluan</span></h1></li></ul>
<p><span class="font3">Perkembangan teknologi yang pesat memberikan dampak yang sangat berguna bagi masyarakat, akan tetapi terdapat juga beberapa dampak negatif dari perkembangan teknologi tersebut, salah satunya adalah penyadapan data. Masalah keamanan data menjadi aspek penting terutama jika data tersebut berisi informasi pribadi. Data merupakan aset yang penting baik bagi individu ataupun sebuah instansi. Dalam beberapa tahun terakhir, terjadi peningkatan signifikan dalam jumlah serangan siber dan pelanggaran data di berbagai sektor, termasuk industri kebugaran [1]. Oleh karena itu, perlunya menerapkan langkah-langkah keamanan yang efektif untuk melindungi data </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3"> menjadi suatu keharusan.</span></p>
<p><span class="font3">Pengamanan file data </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3"> merupakan aspek penting dalam menjaga kerahasiaan informasi pribadi yang disimpan dalam sistem manajemen </span><span class="font3" style="font-style:italic;">gym</span><span class="font3">. Dalam sistem manajemen </span><span class="font3" style="font-style:italic;">gym</span><span class="font3">, biasanya data-data </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3"> disimpan dalam bentuk file yang dapat dengan mudah dilihat dan dibaca. File ini seharusnya memiliki sifat internal yang hanya dapat dilihat oleh pihak internal seperti staf tempat </span><span class="font3" style="font-style:italic;">gym</span><span class="font3">. Permasalahan dapat terjadi jika pihak yang tidak berwenang dapat mengakses dan membaca file data </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3"> dan kemudian menggunakannya untuk melakukan tindakan kriminal. Hal ini dapat terjadi karena instansi tidak memiliki sistem untuk mengamankan file. Untuk menyelesaikan permasalahan ini, dapat dilakukan dengan mengimplementasikan metode kriptografi untuk mengunci isi data dalam sistem pengamanan file sehingga data-data dapat terjamin keamanannya [2].</span></p>
<p><span class="font3">Dalam kriptografi terdapat istilah yang biasa digunakan yakni enkripsi (</span><span class="font3" style="font-style:italic;">encryption</span><span class="font3">) yang merupakan proses untuk mengubah data asli (</span><span class="font3" style="font-style:italic;">plaintext</span><span class="font3">) menjadi data tersandi (</span><span class="font3" style="font-style:italic;">chipertext</span><span class="font3">), dan dekripsi (</span><span class="font3" style="font-style:italic;">decryption</span><span class="font3">) yakni proses untuk mengembalikan data tersandi menjadi data asli [3].</span></p>
<p><span class="font3">Terdapat beberapa algoritma kriptografi yang dapat digunakan untuk mengamankan file, terdapat kunci rahasia untuk mengenkripsi dan dekripsi file sehingga ketika file dienkripsi, file yang dilindungi tidak dapat dilihat oleh sembarang pihak, dan hanya pihak yang memiliki kunci rahasia itu yang dapat mendekripsi dan melihat isi file.</span></p>
<p><span class="font3">Dalam penelitian ini, akan diimplementasikan algoritma kriptografi </span><span class="font3" style="font-style:italic;">Advanced Encryption Standard </span><span class="font3">(</span><span class="font3" style="font-style:italic;">AES</span><span class="font3">) untuk pengamanan file data </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3">. Dengan menggunakan algoritma </span><span class="font3" style="font-style:italic;">AES</span><span class="font3">, informasi yang disimpan dalam sistem manajemen </span><span class="font3" style="font-style:italic;">gym</span><span class="font3"> akan menjadi tidak terbaca dan tidak dapat diakses oleh pihak yang tidak berwenang. Hal ini membantu menjaga kerahasiaan dan integritas data </span><span class="font3" style="font-style:italic;">member</span><span class="font3">, serta mencegah kemungkinan penyalahgunaan atau pencurian informasi pribadi.</span></p>

<h1><a name="bookmark8"></a><span class="font3" style="font-weight:bold;"><a name="bookmark9"></a>2.3. &nbsp;&nbsp;&nbsp;Kriptografi</span></h1></li></ul>
<p><span class="font3">Kriptografi merupakan ilmu untuk menjaga keamanan dan kerahasiaan pesan [5]. Kriptografi adalah ilmu yang mempelajari teknik-teknik untuk melindungi informasi agar tetap rahasia, otentik, dan terjamin keutuhannya. Tujuan utama kriptografi adalah mengamankan komunikasi dan data dari akses yang tidak sah atau perubahan yang tidak diinginkan. Secara umum, kriptografi melibatkan penggunaan algoritma matematis yang kompleks untuk mengubah data asli (</span><span class="font3" style="font-style:italic;">plaintext</span><span class="font3">) menjadi bentuk yang tidak dapat dibaca (</span><span class="font3" style="font-style:italic;">ciphertext</span><span class="font3">) menggunakan kunci kriptografi. Proses ini disebut enkripsi. Untuk membaca kembali isi asli dari </span><span class="font3" style="font-style:italic;">ciphertext</span><span class="font3">, diperlukan kunci yang sesuai untuk melakukan proses dekripsi.</span></p>
<ul style="list-style:none;"><li>
<h1><a name="bookmark10"></a><span class="font3" style="font-weight:bold;"><a name="bookmark11"></a>2.4.</span><span class="font3" style="font-weight:bold;font-style:italic;"> &nbsp;&nbsp;&nbsp;Advanced Encryption Standard</span><span class="font3" style="font-weight:bold;"> (</span><span class="font3" style="font-weight:bold;font-style:italic;">AES</span><span class="font3" style="font-weight:bold;">)</span></h1></li></ul>
<p><span class="font3" style="font-style:italic;">AES</span><span class="font3"> (</span><span class="font3" style="font-style:italic;">Advanced Encryption Standard</span><span class="font3">) adalah sebuah algoritma kriptografi yang digunakan secara luas untuk mengamankan data. Algoritma ini dikenal sebagai standar enkripsi yang kuat dan efisien. </span><span class="font3" style="font-style:italic;">AES</span><span class="font3"> menggunakan pendekatan blok cipher, artinya data dipecah menjadi blok-blok yang sama ukurannya sebelum dienkripsi. Setiap blok data kemudian diolah menggunakan serangkaian operasi matematis yang kompleks, termasuk substitusi byte, pergeseran baris, dan</span></p>
<p><span class="font3">pencampuran kolom. Proses ini dilakukan berulang pada setiap blok data untuk menghasilkan ciphertext.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">a. &nbsp;&nbsp;&nbsp;Proses Enkripsi </span><span class="font3" style="font-style:italic;">AES</span></p></li></ul>
<p><span class="font3">Proses enkripsi pada algoritma AES terdiri dari beberapa jenis perubahan </span><span class="font3" style="font-style:italic;">byte</span><span class="font3">, yakni </span><span class="font3" style="font-style:italic;">SubBytes</span><span class="font3">, </span><span class="font3" style="font-style:italic;">ShiftRows</span><span class="font3">, </span><span class="font3" style="font-style:italic;">MixColumns</span><span class="font3">, dan </span><span class="font3" style="font-style:italic;">AddRoundKey</span><span class="font3">. Operasi yang digunakan dalam </span><span class="font3" style="font-style:italic;">AES</span><span class="font3"> merupakan operasi yang ditentukan dalam domain </span><span class="font3" style="font-style:italic;">finite field</span><span class="font3"> GF (2<sup>8</sup>) dengan polinomial irreducible pembangkit </span><span class="font6">m(x) = x8 + x4 + x3 + x + 1 </span><span class="font3">[6].</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">1. &nbsp;&nbsp;&nbsp;Dalam tahap </span><span class="font3" style="font-style:italic;">AddRoundKey</span><span class="font3">, setiap </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> dalam blok data di-XOR-kan dengan </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> kunci yang sesuai.</span></p></li>
<li>
<p><span class="font3">2. &nbsp;&nbsp;&nbsp;Pada tahap </span><span class="font3" style="font-style:italic;">SubBytes,</span><span class="font3"> Setiap </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> dalam blok data digantikan dengan </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> yang sesuai dalam tabel substitusi (S-Box).</span></p></li>
<li>
<p><span class="font3">3. &nbsp;&nbsp;&nbsp;Pada tahap </span><span class="font3" style="font-style:italic;">ShiftRows</span><span class="font3">, </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> dalam setiap baris blok data digeser ke kiri berulang secara teratur.</span></p></li>
<li>
<p><span class="font3">4. &nbsp;&nbsp;&nbsp;Pada tahap </span><span class="font3" style="font-style:italic;">MixColumns</span><span class="font3">, kolom-kolom dalam blok data diubah menggunakan transformasi matriks.</span></p></li></ul>
<p><span class="font3">Tahap-tahap ini diulang beberapa kali tergantung pada ukuran kunci yang digunakan (128-bit, 192-bit, atau 256-bit).</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">b. &nbsp;&nbsp;&nbsp;Proses Dekripsi </span><span class="font3" style="font-style:italic;">AES</span></p></li></ul>
<p><span class="font3">Proses dekripsi untuk mengembalikan </span><span class="font3" style="font-style:italic;">chipertext</span><span class="font3"> menjadi </span><span class="font3" style="font-style:italic;">plaintext</span><span class="font3"> dapat dilakukan terbalik dengan menerapkan arah yang berlawanan dari proses enkripsi, hal ini sering disebut dengan istilah </span><span class="font3" style="font-style:italic;">inverse chiper</span><span class="font3">. Perubahan byte yang digunakan pada proses ini adalah </span><span class="font3" style="font-style:italic;">InvShiftRows</span><span class="font3">, </span><span class="font3" style="font-style:italic;">InvSubBytes</span><span class="font3">, </span><span class="font3" style="font-style:italic;">InvMixColumns</span><span class="font3">, dan </span><span class="font3" style="font-style:italic;">AddRoundKey</span><span class="font3">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">1.</span><span class="font3" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;InvShiftRows</span><span class="font3"> merupakan operasi yang memulihkan posisi </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> dalam setiap baris blok cipher ke posisi aslinya. Pada tahap enkripsi, </span><span class="font3" style="font-style:italic;">ShiftRows</span><span class="font3"> menggeser </span><span class="font3" style="font-style:italic;">byte-byte</span><span class="font3"> dalam setiap baris ke kiri. Pada tahap dekripsi, </span><span class="font3" style="font-style:italic;">InvShiftRows</span><span class="font3"> menggeser </span><span class="font3" style="font-style:italic;">byte-byte</span><span class="font3"> tersebut kembali ke posisi aslinya dengan pergeseran ke kanan.</span></p></li>
<li>
<p><span class="font3">2.</span><span class="font3" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;InvSubBytes</span><span class="font3"> merupakan operasi yang mengembalikan </span><span class="font3" style="font-style:italic;">byte-byte</span><span class="font3"> dalam blok </span><span class="font3" style="font-style:italic;">cipher</span><span class="font3"> ke nilai asli mereka menggunakan </span><span class="font3" style="font-style:italic;">Inverse S-Box</span><span class="font3">. Pada tahap enkripsi, </span><span class="font3" style="font-style:italic;">SubBytes</span></p></li></ul>
<p><span class="font3">menggantikan setiap </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> dengan </span><span class="font3" style="font-style:italic;">byte</span><span class="font3"> yang sesuai dalam tabel substitusi. Pada tahap dekripsi, </span><span class="font3" style="font-style:italic;">InvSubBytes</span><span class="font3"> mengembalikan </span><span class="font3" style="font-style:italic;">byte-byte</span><span class="font3"> tersebut ke nilai asli mereka dengan menggunakan </span><span class="font3" style="font-style:italic;">Inverse S-Box</span><span class="font3">.</span></p>
<ul style="list-style:none;"><li>
<p><span class="font3">3.</span><span class="font3" style="font-style:italic;"> &nbsp;&nbsp;&nbsp;InvMixColumns</span><span class="font3"> merupakan operasi yang membalikkan transformasi matriks yang dilakukan pada blok </span><span class="font3" style="font-style:italic;">cipher.</span><span class="font3"> Pada tahap enkripsi, </span><span class="font3" style="font-style:italic;">MixColumns</span><span class="font3"> menggabungkan dan mengalikan kolom-kolom dalam blok </span><span class="font3" style="font-style:italic;">cipher</span><span class="font3"> dengan matriks tetap. Pada tahap dekripsi, </span><span class="font3" style="font-style:italic;">InvMixColumns</span><span class="font3"> membalikkan operasi tersebut dengan menggunakan matriks </span><span class="font3" style="font-style:italic;">invers</span><span class="font3">.</span></p></li></ul>

<h1><a name="bookmark26"></a><span class="font3" style="font-weight:bold;"><a name="bookmark27"></a>4. &nbsp;&nbsp;&nbsp;Kesimpulan</span></h1></li></ul>
<p><span class="font3">Berdasarkan penjelasan yang telah dibahas, dapat ditarik kesimpulan bahwa aplikasi berbasis web dari sistem manajemen </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3"> dapat mengimplementasikan algoritma kriptografi Algoritma </span><span class="font3" style="font-style:italic;">Advanced Encryption Standard</span><span class="font3"> (</span><span class="font3" style="font-style:italic;">AES</span><span class="font3">) untuk mengamankan file-file yang berisi data diri para </span><span class="font3" style="font-style:italic;">member gym</span><span class="font3">. Aplikasi ini berhasil melakukan proses enkripsi menjadi format. rda dan mendekripsikannya kembali menjadi bentuk file asli dan tidak mengubah ukuran file aslinya. Sehingga dapat ditunjukkan bahwa penggunaan algoritma AES dalam pengamanan file data member gym memberikan tingkat keamanan yang tinggi dan melindungi informasi sensitif dari pihak yang tidak berwenang. Diharapkan penelitian selanjutnya dapat mengkombinasikan metode kriptografi untuk pengamanan data pada sistem lain, sehingga dapat meningkatkan keamanan file atau data dan mencegah terjadinya penyadapan data oleh pihak yang tidak bertanggung jawab.</span></p>
